- The sphere assertion pins a specific 16-digit value against the *committed
  bytes* of `sphere_r10mm.stl`. Per DECISIONS.md 2026-07-04, expected values are
  computed from on-disk bytes; if that STL is ever re-exported at different
  precision, `expected_values.json` must be recomputed from the new bytes or
  this check will (correctly) go red.
- `STL_FIXTURE_DIR` is still a compile-time absolute path baked in at configure
  time (same trade-off as `MATERIALS_JSON_PATH`): reconfigure if the repo is
  moved/renamed. A clean CI checkout always reconfigures.
- Everything else in handoff 005's "Warnings" section still applies (heuristic
  ASCII/binary format detection; `read_stl_file` does not enforce
  watertightness while `import_stl_file` does; empty mesh is treated as
  not-watertight and rejected). M1.5 voxelizer should consume `import_stl_file`
  (or check the report), not a raw `read_stl_file` result.
