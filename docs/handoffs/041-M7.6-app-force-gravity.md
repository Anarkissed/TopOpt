### Data model (pure value type, headlessly tested) ΓÇö `Sources/TopOptFlows/ForceModel.swift` (new)

group identity/faces/colour), keyed per-group on the group `UUID`. Faithful to
the prototype's `S`, `setGravity`, `mkAnchor`/`mkLoad`, `loadDir`, `fmtW`, the

### Bridge wiring (calls M7.6-core's signatures) ΓÇö `Sources/TopOptFlows/LoadCaseTagger.swift` (new)
Turns the declared load case into the M7.6-core bridge calls: every anchor/load
face is TAGGED (`tag_step_face`, anchorΓåÆFixture / loadΓåÆLoad) **and** MASKED
(`mask_step_face(.frozenSolid, depth N)`) as a passive N-voxel shell so the
optimizer cannot remove the BC surface (D7 / MOD AC). Pending groups are rejected
up front (`.pendingGroup`) so a run never half-applies. Both bridge calls are
injected (default = real `TopOptKit.tagStepFace` / `TopOptKit.maskStepFace`), so
the mapping is spied headlessly. This does not run the solver and does not apply
the traction itself ΓÇö `topopt::traction_loads` is not on the Swift bridge, so the
load force vector is carried forward for M7.7's background run (see NOT-done).

### What each of the six delivered
1. **CameraΓåÆscreen projection seam.** `CameraProjection` (view┬╖projection +
   viewport, with `project(worldPoint) -> CGPoint?`) in OrbitCamera.swift. The
   MTKView Coordinator publishes it via a new `onProjection` closure ΓÇö on mesh
   frame, on every orbit/zoom gesture, and once post-layout (deduped against the
   last value so it can't loop). Pure, headlessly tested.
2. **3D settle (D2).** `MeshRenderer` now applies a model rotation about the mesh
   centre (keeps the camera target fixed) folded into the mesh + id-pass MVP and
   the normal matrix. `beginSettle(to:duration:)` slerps with the prototype's
   ease-out-with-overshoot; the Coordinator un-pauses the view for the ~0.8 s
   animation and re-pauses when it lands. Reduced-motion (`settleAnimated:false`,
   from `@Environment(\.accessibilityReduceMotion)`) snaps (duration 0).
3. **Ground grid + contact shadow.** A separate alpha-blended, depth-tested-but-
   not-depth-writing ground pipeline (optional ΓÇö degrades gracefully like the
   id pass) draws a distance-faded grid + a radial contact-shadow disc on the
   settled floor plane (`showGround` = gravity set Γêº edit phase).
4. **In-scene force arrows (D6).** A SwiftUI `Canvas` overlay projects each load's
   group centroid + world direction (from the settle rotation) and draws the
   prototype's tapered shaft + head in the group colour, tip-at-application-point
   when the force presses in else tail-at-point (consumes
   `ForceModel.arrowTipAtApplicationPoint`); non-active loads show their weight.
5. **Floating overlays (D3/D4/D5 placement).** The Anchor|Load chip / snap row /
   weight pill now `.position` at the active group's projected centroid (via the
   seam), falling back to the foot of the stage when no projection is available
   yet. `ViewerMesh.faceCentroid` (new, tested) supplies the centroid.
6. **Anchor faces tint green in 3D.** The workspace passes role-aware `faceTints`
   (anchors ΓåÆ `ForceModel.tint` green, else palette) into `MetalMeshView`, so the
   3D highlight matches the panel. **Empty-space tap** now drops the pending group:
   the pick seam reports a miss via a new `onMiss` closure ΓåÆ `clearActive`.
