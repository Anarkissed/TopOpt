## Warnings for the next run
- `MATERIALS_JSON_PATH` is a compile-time absolute path baked into the
  `test_materials` binary at configure time. If the repo is moved/renamed after
  configuring, reconfigure (`cmake -S core -B build`) before running the test,
  or the on-disk test will fail to open the file. A clean CI checkout always
  reconfigures, so CI is unaffected.
- The on-disk test loads whatever the human commits at
  `core/src/materials/materials.json`. If that file is later hand-edited into
  an invalid state (e.g. a trailing comma, an out-of-range `z_knockdown`, or a
  resin with `z_knockdown != 1.0`), this test will go red by design — that is
  the loader catching a bad seed file, not a broken test. Fix the JSON.
- The duplicate-name test relies on the JSON parser preserving repeated
  top-level keys in insertion order (it stores object members as a vector, not
  a map) so both `"PLA"` entries reach `parse_materials`. Don't "optimize" the
  parser to dedupe keys silently — that would make the loader accept a
  duplicate instead of rejecting it, and this test would then (correctly) fail.
