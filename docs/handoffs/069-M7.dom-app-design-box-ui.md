## What this is

A workspace tool to define a DESIGN BOX — an axis-aligned volume in model space the
optimizer may grow material into, beyond the imported part — plus optional KEEP-OUT
boxes it must leave empty. The box is drawn as a translucent green volume with
draggable resize/move handles (keep-outs in red), persists on the project, and is
threaded model-space → the core's `design_box`/`keep_out_boxes`. DEFAULT OFF: a
project that never opens the tool passes no box and runs byte-identically to before.

## What changed (all in `/app/`)

### Bridge (consume the dom-core fields — additive, defaulted)
1. **`Sources/TopOptBridge/include/TopOptBridge.hpp`** — `BridgeLoadCase` gains
   `bool has_design_box`, `design_box_min/max_{x,y,z}` (6 doubles), and flattened
   `keep_out_min` / `keep_out_max` (3 doubles per keep-out box). All default to
   off/empty → an unset box is byte-identical to today.
2. **`Sources/TopOptBridge/bridge.cpp`** — in `run_minimize_plastic_loadcase`, when
   `has_design_box` is set: build `topopt::DesignBox` + `keep_out_boxes` and set
   `opts.design_box`. Three seams handled carefully:
   - The M7.anchor-integrity **anchor pad** (a caller `design_mask`) is INCOMPATIBLE
     with `design_box` (minimize_plastic throws if both are set) AND redundant under
     one (expand_design_domain freezes the WHOLE imported part as FrozenSolid, not
     just an N-voxel pad). So the pad is now built only when `!has_design_box`; the
     ladder floor is orthogonal and still applies.
   - **Result-grid metadata.** minimize_plastic takes the PART grid and expands
     INTERNALLY, so every variant's mesh + von-Mises/displacement field + playback
     are indexed to the EXPANDED grid. The reported grid dims/origin/spacing (which
     the app uses to sample the grid-indexed fields at a mesh vertex) must therefore
     describe the expanded grid — rebuilt here with the same deterministic
     `expand_design_domain` call. With no box the result grid IS the part grid.
     *(This was a real bug the end-to-end test caught: field size 65268 vs. reported
     2816 voxels before the fix.)*
   - `<algorithm>` added for `std::min`.

### Swift wrapper + models
3. **`Sources/TopOptKit/TopOptKit.swift`** — new `DesignBoxSpec { min, max: SIMD3<Double> }`;
   `minimizePlasticLoadCase` gains `designBox: DesignBoxSpec? = nil` +
   `keepOutBoxes: [DesignBoxSpec] = []`, pushed into the `BridgeLoadCase` (only when
   a box is present).
4. **`Sources/TopOptFlows/DesignBox.swift`** (NEW) — the pure value-type model:
   - `DesignBoxBounds` (min/max `SIMD3<Float>`, Codable) with `faceCenter`,
     `movingFace` (non-degenerate resize), `translated`, `union`.
   - `DesignBoxModel` (`box: DesignBoxBounds?` + `keepOuts`, Codable): `defaultBox`
     (part bbox grown 40 % → immediate grow room), `enable`/`disable`/`reset`,
     `resizeFace`/`move`, `addKeepOut`/`removeKeepOut`/`resizeKeepOutFace`/`moveKeepOut`,
     and `bridgeBox`/`bridgeKeepOuts` (nil/empty when off → nothing passed).
   - `DesignBoxDrag.axisDelta` — screen-drag → model-space delta along a settled
     axis (camera-independent; 0 for an edge-on axis).
5. **`ProjectModel.swift`** — `@Published var designBox = DesignBoxModel()` (default
   off); wired into `snapshot()` and `restoring:`.
6. **`ProjectStore.swift`** — `ProjectSnapshot.designBox: DesignBoxModel?` (OPTIONAL
   so pre-existing snapshots decode; nil → off).
7. **`RunModel.swift`** — `RunRequest` gains `designBox` + `keepOutBoxes` (part of the
   request identity, so editing the box re-enables Optimize); `bridgeRunner` forwards
   them on the STEP load-case path.
8. **`AppModel.swift`** — `makeRunRequest()` passes `project.designBox.bridgeBox` /
   `.bridgeKeepOuts`.

### UI (device QA)
9. **`MetalMeshView.swift`** — a design-box render pass: `setDesignBoxes` builds
   translucent box faces + bright edges (design green, keep-out red) in MODEL space,
   drawn under the mesh's MVP (settles with the part) via the existing alpha-blended
   ground pipeline + depth-test-no-write state. Threaded through `MeshViewInputs` +
   both platform representables + the Coordinator (rebuilds only when the boxes
   change).
10. **`WorkspacePlaceholder.swift`** — the tool: a "Design Box" chip (top-right) that
    opens/closes it (seeds a grow-room box); a control panel (Reset / Add keep-out /
    per-keep-out remove); and on-scene drag handles — 6 face-resize handles + a centre
    move handle for the design box and each keep-out, driven by `DesignBoxDrag`. Shown
    only in the edit phase once gravity is set (the STEP load-case flow the box feeds).

## Acceptance (per the task)

(a) open a design-box tool → the "Design Box" chip. (b) size a box larger than the
import → the default box already exceeds the part; face handles push further out.
(c) add a keep-out → panel "Add keep-out". (d) optimize → the box threads through the
bridge. (e) see material grow into the box → dom-core physics on the expanded grid
(the end-to-end test proves the grid expands and the fields size to it). Box +
keep-outs persist with the project (relaunch test).

## Notes / follow-ups

- **STEP-only in practice.** The design box feeds the STEP load-case run path
  (`run_minimize_plastic_loadcase`). The UI only surfaces in the edit phase after
  gravity is set, which requires B-rep face normals (STEP) — so an STL project never
  shows the tool and its self-weight path ignores the box. Consistent, no dead UI.
  Wiring the box into the STL self-weight path (`run_minimize_plastic`) is a small
  additive follow-up if STL grow-room is ever wanted.
- **Gizmo feel is device QA.** The translucent box + handle drag/settle + the visible
  material growth are for on-device QA; the tested logic (conversion, resize/move,
  persistence, drag math, grid expansion) is headless.
- **Move handle** slides in the ground plane (model X/Z); vertical sizing is via the
  top/bottom face handles — a deliberate v1 simplification of 2D-drag → 3D-move.
