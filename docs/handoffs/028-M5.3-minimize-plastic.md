## What I did
Added the M5.3 `minimize_plastic` end-to-end driver — the pipeline glue that
ties the earlier milestones together for the self-weight / "minimize plastic"
mode. It is the driver handoffs 026/027 pointed at (the one that "will actually
call check_v3 per variant, feed min_feature_violations and
min_feature_warning_text into each VariantReport, and assemble the JobReport
end-to-end").

New public API — `core/include/topopt/pipeline.hpp` (declaration-only; pulls in
fea/materials/mesh/report/settings/simp/voxel headers):
- `struct MinimizePlasticOptions` — the ladder (`volume_fraction_ladder`,
  descending, default {0.7, 0.5, 0.3}), `margin_stop` (default 1.5), self-weight
  `gravity` + `gravity_direction`, and a shared `SimpOptions simp`.
- `struct MinimizePlasticVariant` — one evaluated rung: its requested fraction,
  the full `SimpOptimizeResult`, the `V3Report` on that output, the assembled
  `VariantReport`, and an `accepted` flag (margin >= margin_stop).
- `struct MinimizePlasticResult` — `evaluated` (every rung run, in ladder
  order), `stopped_on_margin`, and the assembled `JobReport report` (accepted
  variants only).
- `MinimizePlasticResult minimize_plastic(grid, material, material_name, bcs,
  rules, options)`.

Implementation — `core/src/simp/minimize_plastic.cpp` (new TU; Eigen-gated in
CMake alongside simp.cpp, since it drives simp_optimize / simp_compliance).
Per rung, in ladder order:
1. `simp_optimize` (mask-aware overload, all-Active mask so Load/Fixture voxels
   are implicitly FrozenSolid and V3 retention is structural) at that rung's
   volume fraction, under the self-weight load.
2. A final `simp_compliance` solve on the converged physical density to recover
   the displacement field (simp_optimize returns density/compliance, not `u`).
3. Peak stresses over the PRINTED material only (physical density > 0.5), using
   the material's solid modulus: `hex8_stress` per printed voxel → the max von
   Mises and a grid-indexed Voigt field → `max_interlayer_tension(build_dir)`.
4. `compute_stress_margin` (M5.2 locked definition) → worst-case margin.
5. `check_v3` (M3.5) on the output; `recommend_settings` (M5.1) for the margin;
   `min_feature_warning_text` (M5.2b) for the V3 min-feature count → one
   `VariantReport`.
6. Accept (append to the JobReport) while margin >= margin_stop; on the first
   rung below margin_stop, record it in `evaluated`, set `stopped_on_margin`,
   and STOP (lighter rungs are not run).

Key design choices and WHY (all documented in pipeline.hpp):
- **Self-weight is the design load**, computed once with `self_weight_loads` on
  the original solid grid and held fixed across rungs. The minimum-compliance
  topology is invariant to the load magnitude, so a fixed load vector gives
  comparable variants; a density-coupled self-weight iteration is out of scope
  for this milestone.
- **The build/print orientation is an INPUT** (`gravity_direction`; the reported
  build direction is its unit negation). The driver does not re-run the M4.4
  orientation search — the app / M4.4 scorer chooses the orientation and feeds
  it here, keeping the self-weight direction and the reported orientation
  consistent. The margin's interlayer term uses that build direction.
- **Peak stress is taken over printed voxels only** (physical density > 0.5),
  with the solid modulus — the stress in the material that actually prints; the
  penalized-soft void voxels are excluded.
- **Fixture retention is structural** (mask-aware optimize, Load/Fixture
  implicitly FrozenSolid, M3.7), so the §7 V3 retention gate holds by
  construction.
- **The driver validates its own inputs** (ladder non-empty / in (0,1] /
  strictly descending; margin_stop finite >= 0; gravity finite > 0; non-zero
  direction) with `std::invalid_argument`, matching simp_optimize's convention;
  material / BC / margin errors propagate from the callees.

### How the tests are set up and why they are trustworthy
`minimize_plastic` (new, `tests/validation/test_minimize_plastic.cpp`,
Eigen-gated, public API only, **74 checks**). It uses a synthetic in-code FDM
material with a low yield strength (real PLA's 55 MPa yields an astronomically
large self-weight margin for a cm-scale part, so the ladder would never cross
1.5 — the synthetic material is an in-code struct, NOT an edit to materials.json,
like the synthetic variants in `test_report`). No golden numbers are pinned.
- **A — accept-all** (tiny gravity 1e-6): every rung's margin ≫ 1.5, whole
  ladder accepted, no stop; margins measured here are monotone decreasing in
  volume fraction and are reused to calibrate scenario C.
- **B — immediate stop** (gravity 1e6): even the densest rung is too weak → one
  rung evaluated, rejected, empty (but still schema-valid) report.
- **C — mixed walk** (gravity DERIVED from A's margins): because the margin is a
  ratio and the min-compliance topology is load-magnitude-invariant, each rung's
  margin scales exactly as 1/gravity; the gravity is set to the geometric mean of
  the last two rungs' margins over margin_stop, placing the fixed 1.5 threshold
  strictly between them — so the driver accepts the two stronger rungs and stops
  at the too-weak final rung (a genuine mid-ladder stop, not a guessed gravity).
- A shared **stop invariant** check (accepted prefix all >= stop; the terminal
  rejected rung < stop; nothing evaluated past it; report == accepted rungs;
  accepted volume fractions descend; assembled JobReport validates), applied to
  A/B/C.
- **§7 V3 on every accepted output** (watertight + single component + Fixture
  retention; report min-feature count mirrors the V3 count).
- **D — determinism** (identical inputs → byte-identical report JSON).
- **E — settings coupling** (each variant's settings equal `recommend_settings`
  for that variant's margin — the driver wires the M5.1 engine, not a hardcode).
- **F — argument validation** (empty / non-descending / non-strict / out-of-range
  / zero ladder fraction, zero gravity, negative margin_stop, zero direction all
  throw `std::invalid_argument`).

## Warnings for the next run
- **Units are the caller's responsibility.** The margin is a ratio, so the
  ABSOLUTE `gravity` scale sets where the ladder crosses `margin_stop`. For a
  physical margin the caller must pass consistent units (E/yield in MPa, lengths
  in mm ⇒ forces in N ⇒ `density * gravity` in N/mm³). The test picks gravity to
  place the crossing; it is not a claim about real PLA under Earth gravity.
- **The self-weight load is the original solid's weight, fixed across rungs.**
  A lighter variant is analysed under the same load as the full part (see modeling
  note). This is deliberate and monotone; if the maintainer wants a density-
  coupled self-weight, that is a new task.
- **`stopped_on_margin == false` can mean "the whole ladder was strong enough"**
  (accept-all) — the report then holds every ladder rung. An empty report means
  even the densest rung was too weak (immediate stop). Both are valid outcomes a
  consumer must handle.
- **`evaluated` includes the rejected terminal rung; `report.variants` does
  not.** The rejected rung is kept in `evaluated` (with `accepted == false`) for
  transparency, but only accepted rungs reach the JobReport.
- **The min-feature warnings fire heavily on the coarse 20×4×5 test grid** (58 /
  138 / 90 regions): SIMP designs at low resolution are full of sub-2-voxel
  features. This exercises the M5.2b path; it is report-only and does not gate.
- **The margin only stays monotone in volume fraction for well-behaved parts.**
  On a constrained L-domain I first tried, the densest rung under-filled and had
  the LOWEST margin (non-monotone), which makes a descending-ladder mid-ladder
  stop impossible. The test uses a clamped-root cantilever (like test_v3 /
  test_variants) where lower volume fraction ⇒ weaker ⇒ lower margin.
