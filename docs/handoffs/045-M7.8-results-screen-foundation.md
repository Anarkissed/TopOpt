### What I added
Still `/app/`-only, additive; no `/core/`, fixtures, `materials.json`, CI, or
`Package.swift`; no existing assertion changed; `<stdexcept>` grep clean.

- **Bridge arrays** (`TopOptBridge.hpp`/`bridge.cpp`): `OptimizeVariant` now also
  carries the extracted variant isosurface (`mesh_vertices`/`mesh_indices`, from
  `v.mesh()`) and the M7.0b per-voxel `von_mises_field`; `OptimizeResult` carries
  the grid metadata (`grid_nx/ny/nz`, `grid_origin_*`, `spacing`) needed to sample
  the field at a mesh vertex. Swift `OptimizeVariant`/`OptimizeOutcome` mirror them.

  core index `(k*ny+j)*nx+i`, with clamping. `ResultsModel` exposes `selectedMesh`
  (variant `ViewerMesh`), `selectedStressField`, and `stressTints(for:field:)` —
  per-flat-vertex colors sampled against the shared scale via the existing
  `stressColor` ramp.
- **Metal viewer** (`MetalMeshView.swift`): additive optional inputs
  `stressTints: [SIMD4<Float>]?` (per-flat-vertex, uploaded into the existing tint
  buffer — the shader already mixes it over the clay) and `reveal: Float` (a
  model-Y clip driven by the scrub, via an isolated fragment `constant float4`
  bound at buffer 0 — the shared `ViewerUniforms` and the id/ground passes are
  untouched, so edit-mode is unchanged; default `reveal = 1` disables the clip).
- **`ResultsScreen`**: now hosts its OWN opaque `MetalMeshView` (the workspace
  viewer stays untouched underneath) showing `model.selectedMesh`, stress-colored
  when `stressOn`, morphing with `playT`.
