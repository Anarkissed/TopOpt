## Blocked

### M7.viz.3 needs a per-vertex displacement field on the bridge

**What is missing:** a per-vertex (or per-node) **displacement vector** field for
the displayed variant, exposed through the bridge. The bridge currently exposes
only the von Mises **scalar** field (`OptimizeVariant.von_mises_field`,
`TopOptBridge.hpp:164`) plus geometry and isosurface keyframes. A scalar cannot
drive vertex motion, so the flex animation cannot be built.

**Where the data already is (so the core track's add is a plumb-through, not new
physics):** the linear-elastic solve already produces the nodal displacement
field — `FeaSolution` (DOF-ordered, size `3*fea_node_count`, `fea.hpp:183`),
retained as `SimpOptimizeResult.solution` (`simp.hpp:53`) and reachable in core as
`MinimizePlasticVariant.optimization.solution`. Nothing new needs solving; the
physics is done. It just is not surfaced on the M7.0b visualization block nor
forwarded across the bridge.

**Whose change this is:** the CORE track, per the interface freeze
(ARCHITECTURE §8 rule 10; DECISIONS 2026-07-11 concurrent mode — app track is
`/app/`-only, core track owns `/core/` + the bridge). NOT the app track. This
handoff is the requested Blocked hand-off; the app track does not add it.

**Suggested shape of the core-track deliverable (for whoever picks it up — not
prescriptive):**
- Expose the displayed variant's displacement as vectors the app can apply to the
  **variant display mesh vertices** (`OptimizeVariant.mesh_vertices`). The core
  solution is DOF-ordered per FEA node, so the core/bridge side should provide
  either (a) per-mesh-vertex displacement (xyz per `mesh_vertices` entry, already
  sampled from the nodal field at each vertex — mirrors how `von_mises_field` is
  documented to be sampled at a vertex), or (b) the per-node field plus the
  node-grid mapping needed to sample it, documented the way `von_mises_field`'s
  grid sampling is documented at `TopOptBridge.hpp:186-189`. Option (a) is the
  smaller app-side surface and matches the existing per-vertex color pattern.
- Populate it for every evaluated non-cancelled rung (same lifecycle as
  `von_mises_field`), empty for a cancelled rung.
- Add core/bridge tests (nodal field → per-vertex sampling correctness; consistency
  with the variant mesh; empty on cancel), consistent with the M7.0b test pattern.

Once that field is merged onto the bridge, M7.viz.3 is unblocked and this app-track
task can proceed exactly as specified (scale 50–100x, animate rest→full→rest loop,
reduced-motion = static full-deflection frame, Metal vertex animation), consuming
the new field only.
