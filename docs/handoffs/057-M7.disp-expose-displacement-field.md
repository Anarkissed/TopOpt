### Core API — `core/include/topopt/pipeline.hpp`
`MinimizePlasticVariant` gains one member, in the M7.0b visualization block:
- `std::vector<double> displacement_field;` — per-NODE, DOF-ordered, size
  `3*fea_node_count(grid)`; `[3n,3n+1,3n+2] = (ux,uy,uz)` of node n in model
  units (mm). It is the nodal displacement of the SAME final penalized solve
  that produces `von_mises_field` (`SimpCompliance::solution`), retained per
  node. Zero on nodes attached only to non-printed voxels (physical density <=
  the M3.5 iso 0.5), mirroring how `von_mises_field` is zero off the printed
  set. Empty on a cancelled rung.

The `MinimizePlasticResult::cancelled` doc contract was extended to name
`displacement_field` among the visualization fields left default on a cancelled
rung.

### Core driver — `core/src/simp/minimize_plastic.cpp`
In the existing per-variant stress loop (the SAME loop that fills
`von_mises_field` from the final penalized solve `sc = simp_compliance(...)`; no
new solve is added):
- While iterating printed voxels (`physical density > kIso`), mark each printed
  voxel's eight corner nodes (`fea_element_nodes`) in a `node_printed` bitmap.
- After the loop, `displacement_field` is assigned size `3*fea_node_count(grid)`
  and, for every printed node, set element-for-element to `sc.solution.at(n,c)`;
  nodes attached only to non-printed voxels stay zero.
The cancelled path is unchanged: a cancelled rung `break`s before this analysis
block, so `displacement_field` stays default-empty exactly like
`von_mises_field` (pipeline.hpp cancel contract).

### Bridge (forward the one new field; TRACK header exception)

narrowing for Metal), no signature changed:

- `app/TopOptKit/Sources/TopOptBridge/bridge.cpp`: `to_optimize_variant` assigns

## Warnings for the next run
- **`displacement_field` is per-NODE (size 3*node_count), while
  `von_mises_field` is per-VOXEL (size voxel_count).** A viewer must not index
  one with the other's stride. Node n is grid corner (a,b,c) at
  `(c*(ny+1)+b)*(nx+1)+a`; voxel index is `(k*ny+j)*nx+i`. The bridge doc
  comment spells out the node indexing against `grid_nx/ny/nz`.
- **A node is "printed" (nonzero) iff it is a corner of at least one printed
  voxel (density > 0.5).** So a printed node adjacent to void still carries its
  full solved displacement; only nodes with NO printed incident voxel are zero.
  This is deliberately more permissive than the per-voxel field: the mesh
  vertices M7.viz.3 animates sit on printed-voxel corners, so they are always in
  the nonzero set.
- **Clamped (Fixture) nodes read exactly 0** in the field (Dirichlet BCs), even
  though they are printed — that is the correct rest position for the animation,
  not a masking artifact.
- **Exact element-for-element equality relies on `simp_compliance` determinism**
  (the same property Scenario D's byte-identical report depends on). If a future
  change makes the penalized solve non-deterministic (threaded reductions, etc.),
  `check_displacement_field`'s `==` would need a tolerance — but so would several
  existing determinism checks.
- **Bridge field is `float`** (narrowed from the core `double`), matching
  `von_mises_field`. Fine for rendering/animation; a consumer needing full
  double precision must read the core field, not the bridge field.
