# Handoff 058 — M7.viz.3: flex animation (the "wow")

## Task
M7.viz.3 (TRACK app / M7-VIZ block): animate the FEA displacement solution — scale
the per-vertex displacement 50–100× (user-adjustable, default tuned for phone
legibility) and animate the mesh rest → full deflection → back, looping. Reduced-
motion disables the loop and shows a static full-deflection frame. NOT a physics
simulation — the displacement is already solved (M7.disp); this only DRAWS it. Plus
the consumer's job: PERSIST the displacement field through OutcomeStore so a saved-
then-reloaded project animates without re-optimizing.

The bridge field already exists: M7.disp (merged, handoff 057) exposed
`OptimizeVariant.displacementField: [Float]` (DOF-ordered, per-NODE, size
`3*(gridNx+1)*(gridNy+1)*(gridNz+1)`, mm; zero on non-printed nodes; empty on a
cancelled rung) and `TopOptKit.convertOutcome` already maps it. So this run does NOT
touch `/core/` or the bridge — it consumes the existing field (interface freeze
held). Verification is `xcodebuild test` + maintainer device QA (DECISIONS
2026-07-09), not Linux ctest.

### Pure, headlessly-tested logic — `ResultsModel.swift`
- **`DisplacementField`** (new): the per-NODE sibling of `StressField`. `nx/ny/nz`
  are VOXEL dims, so there are `(nx+1)*(ny+1)*(nz+1)` nodes; `displacement(at:)` does
  a nearest-NODE lookup (`.rounded()`, index `(c*(ny+1)+b)*(nx+1)+a`) — the
  companion of `StressField.value(at:)`'s nearest-VOXEL lookup. **CRITICAL indexing
  difference is respected**: displacement is per-node (stride 3·nodeCount), von Mises
  is per-voxel (stride voxelCount) — never cross-indexed. `isEmpty` also guards a
  short/ragged field (`values.count < 3*nodeCount`) so a partial field can't OOB.
- **`FlexAnimation`** (new pure enum): the exaggeration band (`min 50`, `max 100`,
  `default 60`) + `clampExaggeration`, the rest→full→rest `amplitude(phase:)` (raised
  cosine `0.5 - 0.5·cos(2π·phase)`: 0 at phase 0, 1 at 0.5, 0 at 1, zero-slope
  turnarounds so the loop has no seam), and `displacedPosition(base:displacement:
  exaggeration:amplitude:)` = the exact shader math (`base + exaggeration·amplitude·d`).
- **`ResultsModel`** flex state: `flexOn`, `flexPhase` (loop position, wrapped),
  `flexExaggeration` (clamped via `setFlexExaggeration`), `hasFlex`,
  `selectedDisplacementField`, `toggleFlex` (resets phase to rest on off),
  `advanceFlex` (wraps in [0,1)), `flexAmplitude(reduceMotion:)` (reduced-motion pins
  amplitude 1 = static full deflection; else follows the phase; 0 when off),
  `flexScale(reduceMotion:)` = `exaggeration·amplitude`, and
  `flexDisplacements(for:field:)` — per-flat-vertex displacement vectors sampled once
  and cached on `selectedIndex` (invalidated in `apply`, so it is O(N) per variant,
  not per frame). `select()` also resets `flexPhase` so a new variant starts at rest.

### Metal vertex animation — `MetalMeshView.swift`
- Viewer vertex shader gains `constant packed_float3* disp [[buffer(3)]]` +
  `uint vid` and applies `p = position + u.flex.x · disp[vid]` before the MVP;
  `Uniforms`/`ViewerUniforms` gain a trailing `float4 flex` (`.x` = scale). Appended
  AFTER `mvp`/`normalMatrix`, so the id pass (its `Uniforms` reads only that 128-byte
  prefix; the `setVertexBytes` now supplies 144) is byte-unaffected and needs no
  buffer(3).
- `MeshRenderer`: a `flexBuffer` (per-flat-vertex packed float3) is ALWAYS bound at
  buffer(3) — zero-filled in `setMesh`, so the static workspace draw is unchanged and
  scale 0 is a byte-identical rest frame. `setFlexDisplacements` (guards
  `count == vertexDrawCount*3`), `setFlexScale`, `resetFlex` (re-zeros so a stale
  variant can't leak). `makeUniforms` writes `flex.x = flexScale`.
- `MeshViewInputs` + both platform inits gain `flexDisplacements` / `flexScale`; the
  Coordinator uploads the displacement buffer on mesh change / first arrival (it only
  depends on mesh+field, not phase) and pushes the per-frame `flexScale` uniform
  (drives redraw). Turning flex off calls `resetFlex`.

### Screen wiring — `ResultsScreen.swift`
- A "Flex" toggle in the top-right row, shown only when `model.hasFlex` (a
  cancelled/legacy variant has no field). While flexing, the stage shows the final
  variant mesh fully formed (`reveal = 1`, not the history morph), displaced by the
  field.
- A top-leading "Deflection" panel with the 50–100× exaggeration slider + an honest
  "not to scale" caption (under reduced-motion: "showing full deflection").
- The existing ~30 fps ticker advances the flex loop while `flexOn && !reduceMotion`
  (reduced-motion holds the static frame — no advance, `flexScale` uses amplitude 1).
- Feeds `flexDisplacements` (cached) + `flexScale(reduceMotion:)` to `MetalMeshView`.

### Persistence — `OutcomeStore.swift` (the consumer's "PERSIST IT" job)
`VariantDTO` gains `displacementField: Data?`, packed in `dto(from:)` beside
`vonMisesField` and unpacked in `outcome(from:)`. Made **optional** deliberately: a
persist-c blob written before this field existed has no key, so a non-optional field
would fail the WHOLE outcome decode (every already-saved project loses its cached
result on this update). Optional → a legacy blob decodes with an empty flex field
(no animation until a re-run), the rest of the outcome intact.

## Warnings for the next run
- **Per-NODE vs per-VOXEL indexing.** `displacementField` is per-node (size
  `3·(nx+1)(ny+1)(nz+1)`); `vonMisesField` is per-voxel (size `nx·ny·nz`).
  `DisplacementField.displacement(at:)` rounds to the nearest NODE; `StressField`
  floors to the nearest VOXEL. Do not swap them.
- **Nearest-node sampling is intentional.** Marching-cubes vertices sit on printed-
  voxel corners (M7.disp handoff), which are grid nodes, so nearest-node lands on the
  exact node in practice. This is a drawing exaggerated 50–100×; sub-voxel sampling
  accuracy is irrelevant.
- **`flexDisplacements` is cached on `selectedIndex`** (invalidated in `apply`), so it
  samples O(N) once per variant, not per frame. If a future change makes the selected
  mesh vary without changing `selectedIndex`, invalidate `flexCache` there too.
- **The DTO field is `Data?` for backward compat.** Do not "tidy" it to non-optional —
  that would make every pre-M7.viz.3 saved project fail to decode (see
  `testDecodesLegacyBlobWithoutDisplacementField`). Same discipline the other heavy
  arrays would need if they post-dated shipped data.
- **Env / vendor staleness (unchanged from handoffs 055/056).** A fresh worktree has no
  `vendor/`; run `./app/scripts/build_core.sh` to build `TopOptCore.xcframework` against
  the worktree core, or the app tests fail on a `SimpOptions`-layout mismatch. Keep
  `occt-frameworks.generated.json` absent for the macOS-test path.
