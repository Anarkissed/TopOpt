## What I did
Implemented the isotropic 8-node trilinear hexahedral element stiffness matrix
and wired a new `hex_element` ctest that asserts against the committed reference
fixture.

Public API — `core/include/topopt/fea.hpp` (new, the FEA module's public header,
ARCHITECTURE §3/§4):
- `struct Hex8Stiffness`: a 24×24 row-major `std::array<double,576>` with an
  `operator()(row,col)` accessor and `kNodes=8` / `kDof=24` constants.
- `Hex8Stiffness hex8_stiffness(double youngs_modulus, double poisson, double
  element_size)`: the element stiffness for a cubic voxel. Node order is
  bottom-CCW-then-top-CCW (n0(0,0,0)…n7(0,1,1)), DOF order node-major
  interleaved, Voigt order `[xx,yy,zz,gxy,gyz,gzx]` with engineering shear —
  all three matching the fixture's documented conventions exactly. Throws
  `std::invalid_argument` for E≤0, element_size≤0, or ν∉(-1,0.5).

Implementation — `core/src/fea/hex_element.cpp` (new, in `fea/` per §3):
- Isotropic constitutive matrix D built from E, ν (Voigt, engineering shear).
- Full 2×2×2 Gauss quadrature (points ±1/√3, weights 1) — exact for the
  trilinear hex, which is what the fixture used.
- Isoparametric map for a cubic voxel of edge h: x=(h/2)(ξ+1) ⇒ J=(h/2)I,
  detJ=(h/2)³, dN/dx=(2/h)dN/dξ. For each Gauss point it forms the 6×24 B
  matrix and accumulates Kᵉ += Bᵀ D B · detJ. This yields the scaling law
  K(E,h)=E·h·K(1,1) the fixture documents.
- Pure C++/std only — **no Eigen, no OCCT**. Eigen (ARCHITECTURE §4's linear-
  algebra choice) is for the global sparse system and CG solver (M2.2/M2.3);
  the element matrix is small dense arithmetic that needs neither, and keeping
  it dependency-free lets it build in every configuration (Eigen is not
  installed on the local macOS toolchain; see "Warnings"). M2.2 can wrap this
  dense array into Eigen when it assembles the global system.

Build — `core/CMakeLists.txt`:
- Added `src/fea/hex_element.cpp` to the `topopt` library (unconditionally, like
  `voxelize.cpp`; it has no optional-dependency gate).
- Added the `test_fea` executable + `add_test(NAME hex_element)`, injecting
  `FEA_FIXTURE_DIR` (same compile-definition pattern as the other fixture
  tests). Not inside the `if(OpenCASCADE_FOUND)` block, so `hex_element` runs in
  every configuration — including the OCCT-free local build and CI. CI's
  existing `ctest` step picks it up with no CI-file change.

Tests — `core/tests/unit/test_fea.cpp` (new), registered as ctest `hex_element`
(same self-contained CHECK harness as the other unit tests, no framework, §4).
It parses the reference fixture with a small **test-local** JSON reader (a
trimmed object/array/string/number subset parser; not a public library API — the
production JSON parser is materials-internal and unrelated), then drives
`hex8_stiffness` and asserts against the fixture values.

## What I did NOT do

  directly. That check in the fixture's `required_properties` needs an
  eigensolver (Eigen), which M2.1's element math does not otherwise require and
  which is not installed locally. Instead the test proves 3 rigid-translation
  zero modes explicitly (K·t=0) and matches the full 24×24 reference entrywise;
  the reference was independently verified by the maintainer to have exactly 6
  zero eigenvalues (3 translations + 3 linearized rotations), so an exact
  entrywise match transitively guarantees that property. An explicit eigen-count

- Did not introduce Eigen or any global-assembly / boundary-condition / load
  machinery. That is M2.2 (assembly + Dirichlet BCs + loads) and M2.3 (CG

## Warnings for the next run
- **Node / DOF / Voigt conventions are load-bearing.** The element uses
  bottom-CCW-then-top-CCW node order, node-major interleaved DOFs, and Voigt
  `[xx,yy,zz,gxy,gyz,gzx]` with *engineering* shear — chosen to match the
  fixture. M2.2's global assembly must map voxel corners to these node indices
  in the same order, or the assembled system will be silently wrong even though
  each element matches. The conventions are documented at the top of `fea.hpp`.
- **`element_size` must be one cubic voxel edge.** The voxelizer (M1.5) produces
  cubic voxels with a single `spacing`; pass that as `element_size`. The matrix
  scales as E·h·K(1,1), so a per-voxel modulus (SIMP, M3) multiplies the same
  K_ref — no re-integration needed per element, only a scalar multiply.
- **ν = 0.5 is rejected** (incompressible → division by 1−2ν). The seed
  materials are ν≤0.4 so this does not bite, but a caller must not pass 0.5.
- **Eigen is deferred to M2.2/M2.3 by choice, not oversight.** If a reviewer
  expects the element to already use Eigen types (§4), note that §4 governs the
  linear-algebra/solver layer (the global sparse system + CG); the local
  element integrand is plain arithmetic and Eigen is not installed on the local
  toolchain, so keeping the element Eigen-free preserves the dependency-light
  local build. The dense 24×24 array converts trivially to an Eigen block later.
- `FEA_FIXTURE_DIR` is a compile-time absolute path baked in at configure time
  (same trade-off as the other tests): reconfigure if the repo is moved. A clean
  CI checkout always reconfigures.
