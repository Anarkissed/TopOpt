## What I did
- Diagnosed CI run 28627352359 (first run after Nadim created the GitHub remote
  and pushed): `vcpkg install` failed while building the `gperf` port — a
  transitive dependency of `opencascade` (via fontconfig). gperf's portfile
  requires `autoconf-archive` from the **system** package manager, and the
  workflow's apt list had `autoconf automake libtool bison` but not
  `autoconf-archive`. The log-tail errors "CMake was unable to find Ninja" /
  "CMAKE_CXX_COMPILER not set" were cascade noise from the aborted configure
  (they occur inside CMakeDetermineSystem after the toolchain bails), not
  separate problems — ninja-build was already installed.
- Fix: added `autoconf-archive` to the apt install line in
  `.github/workflows/ci.yml`. One-word diff; strict extension of CI.
- A direct push to `main` was denied by the session permission policy, so the
  fix went up as branch `fix/m1.1-ci-autoconf-archive` → **PR #1**
  (https://github.com/Anarkissed/TopOpt/pull/1). The workflow's
  `pull_request` trigger ran the full pipeline on the PR.
- CI run 28628959534 on the PR: **success** in 2h23m14s (uncached — built OCCT
  and all vcpkg deps from source). The vcpkg binary cache was saved (~331 MB,
  key `vcpkg-x64-linux-dynamic-3110eadb…`), so subsequent runs should be minutes.
- Checked the M1.1 box in ROADMAP.md (the only ROADMAP edit).
