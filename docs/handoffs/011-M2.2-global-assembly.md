## What I did
Added the global linear-elastic FEA system on a voxel grid: node numbering,
assembly of every solid voxel's Hex8 element into a sparse global stiffness,
Dirichlet BCs (per-DOF, incl. non-zero prescribed displacement), nodal loads,
and a direct solve. Then verified it on an exact 2×1×1 uniaxial-tension problem.

Public API — `core/include/topopt/fea.hpp` (extended; still **Eigen-free**):
- Node model documented in the header — nodes at voxel corners, a grid of
  nx·ny·nz voxels has (nx+1)(ny+1)(nz+1) nodes; node (a,b,c) id
  `(c*(ny+1)+b)*(nx+1)+a`, DOF `3*node+comp` (comp 0/1/2 = x/y/z).
- `fea_node_count`, `fea_node_index`, `fea_element_nodes` (the eight corner
  nodes of a voxel in the M2.1 Hex8 order — bottom-CCW then top-CCW, so global
  assembly matches the element matrix exactly).
- `fea_tagged_nodes(grid, VoxelTag)`: distinct, sorted corner nodes of every
  voxel carrying a tag. This is the bridge from M1.6 face tags (Load / Fixture)
  to the DOFs a caller loads or constrains — "loads on tagged voxels".
- `struct DirichletBC {node, component, value}` — fix one DOF to a prescribed
  displacement (usually 0; non-zero supported).
- `struct NodalLoad {node, component, value}` — a force on one DOF. A point load
  is one entry; a distributed load is the same total split across several nodes.
- `struct FeaSolution {std::vector<double> u; double at(node,comp);}`.
- `fea_solve(grid, E, nu, bcs, loads)` — assemble + apply + solve → FeaSolution.

The public API returns `std::vector<double>` and takes plain structs, so
`fea.hpp` needs no Eigen (the element-only `hex_element` test keeps building
without Eigen). Eigen is an implementation detail of the .cpp below.

Implementation — `core/src/fea/assembly.cpp` (new, in `fea/` per §3):
- Assembles the global sparse stiffness from `hex8_stiffness(E, nu, grid.spacing)`
  (identical per voxel — uniform material, cubic voxel) over all solid voxels
  (tag != Empty) via Eigen triplets.
- Loads scatter into a dense RHS; range-checked.
- Dirichlet BCs go to the RHS by the standard reduction: solve
  `K_ff u_f = f_f − (K u_prescribed)_f` on the free DOFs, using a selection
  matrix `P` so `K_ff = P K Pᵀ`. Prescribed non-zero displacements are handled.
- Solves `K_ff` with **Eigen `SimplicialLDLT`** — ARCHITECTURE §4 explicitly
  allows SimplicialLDLT for small problems; the CG-with-Jacobi solver for large
  voxel systems is the separate M2.3 task, so I did not build it here (§8: no
  speculative abstraction).
- Singularity is detected from the factorisation's pivot diagonal
  (`solver.vectorD()`): a rigid-body (zero-energy) mode shows up as a near-zero
  pivot **regardless of whether the load is orthogonal to that mode**. A pure
  residual check is insufficient — a consistent-but-singular system (e.g. only
  the x=0 plane pinned in x, loaded purely in x) leaves a small residual yet a
  non-unique answer; the pivot check catches it. `solver.info()` and a
  finiteness check guard the true-failure paths. Throws `std::invalid_argument`
  on singular systems and out-of-range BC/load indices.

(self-contained CHECK harness, no framework, §4; public API only).

Why the 2×1×1 is solved *exactly*: uniaxial tension held by three symmetry
planes (x=0: uₓ=0, y=0: u_y=0, z=0: u_z=0) and driven by a consistent nodal
load on the +x face. Those symmetry planes carry zero transverse reaction, so
the continuum solution is the uniform-strain field uₓ=εx, u_y=−νεy, u_z=−νεz
with ε = F/(A·E). A trilinear hexahedron represents a linear displacement /
constant strain field to machine precision (constant-strain patch test), so the
FE solution equals that field exactly and every one of the 36 DOFs is checked
against a hand formula. With E=100, ν=0.25, h=1, A=1, total tip force F=1
(0.25 at each of 4 tip nodes): ε=0.01, tip uₓ=0.02, mid uₓ=0.01, lateral
contraction −0.0025.

## What I did NOT do
- **Did not build the CG + Jacobi solver.** That is M2.3 (64³ in CI budget).
  M2.2 solves the small system with a direct `SimplicialLDLT`, which §4 sanctions
  for small problems. M2.3 can swap the solver behind `fea_solve` or add a
  variant without touching the assembly/BC/load code.
- **Did not add stress / strain / von-Mises recovery** from the displacement
  field. That is M2.4. `fea_solve` returns displacements only.
- **Did not add reaction-force output** or expose the assembled `K` (the public
  API stays Eigen-free). Assembly is verified end-to-end through the exact
  solve, linearity, and prescribed-displacement checks rather than by inspecting
  `K` directly.
- **Did not implement a "fully fix a Fixture voxel" or "traction over a tagged
  face" convenience.** BCs are per-DOF (a symmetry plane fixes one component on
  a set of nodes — a whole-voxel fix would over-constrain the exact test), and
  loads are per-node. `fea_tagged_nodes` gives the node set; selecting a face or
  a component is the caller's concern (it belongs to the load-direction logic of
  M4 / self-weight, not here). §8: no speculative abstraction.
- **Did not handle voxel grids with nodes attached to no element** (interior
  voids). For such a grid an unattached free node yields a zero pivot and
  `fea_solve` throws "singular"; the 2×1×1 (and any fully-solid grid) has no such
  nodes. SIMP void regions (M3) will need those DOFs pinned or filtered — a
  later concern, flagged here.
- Did not touch `/app/`, `ARCHITECTURE.md`, `materials.json`, CI config, any
  fixture, or any existing test/assertion/tolerance. The ROADMAP change is the
  single M2.2 box.

## Warnings for the next run
- **`fea_solve` is a direct dense-ish solve; do not point it at 64³.** The
  selection-matrix reduction and `SimplicialLDLT` are for the small M2.2
  problem. M2.3 must add the CG/Jacobi path for large grids (that is its whole
  point) — extend or branch inside `fea_solve`, keeping the Eigen-free public
  signature.
- **BCs are per-DOF, loads are per-node — deliberately.** "Loads on tagged
  voxels" is realised through `fea_tagged_nodes` + explicit `NodalLoad`s, not a
  whole-voxel load helper. The exact patch test needs the load on only the outer
  (+x) face of the tagged voxel and symmetry (single-component) BCs; a
  whole-voxel fix or an all-8-node voxel load would break the clean uniform
  field. Any caller that "fixes a fixture voxel" must decide which components.
- **Node order is load-bearing and matches M2.1.** `fea_element_nodes` returns
  the Hex8 bottom-CCW-then-top-CCW order; the DOF map is `3*node+comp`. Assembly
  and the element matrix must agree or the global system is silently wrong — the
  exact patch test is the guard.
- **Singularity is detected via the LDLT pivot diagonal**, not the residual. If
  M2.3 replaces the solver with CG, it needs its own rank/convergence guard —
  CG on a singular consistent system converges to *a* solution silently.
- **Eigen 5.0.1 quirk:** chaining `(P*K*P.transpose()).pruned()` failed to
  compile (a `sparse_sparse_product_with_pruning_selector` template error);
  materialising the products into explicit `SparseMatrix` temporaries
  (`K*Pᵀ` then `P*(…)`) compiles cleanly. Keep the products un-chained.
- Local Eigen is homebrew 5.0.1; CI uses the vcpkg-pinned Eigen. `SimplicialLDLT`
  / `vectorD()` are stable across both.
