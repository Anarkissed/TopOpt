## What this is

Today the optimizer can only REMOVE material from the imported part (the Active
region = the import). This task lets it ADD material BEYOND the imported part —
grow ribs/gussets/buttresses into empty space along the load path — by voxelizing
a user-defined DESIGN VOLUME (an axis-aligned box in model space) and expanding
the Active region into it. Pure physics, FEA-verified, no ML. The mechanism is
the existing M3.7 mask (FrozenSolid=keep, FrozenVoid=exclude, Active=optimizer's
choice); this task expands the ACTIVE region beyond the import.

## What changed (core only)

1. **`core/include/topopt/voxel.hpp`** — new pure-geometry API:
   - `struct DesignBox { Vec3 min, max; }` — axis-aligned design/keep-out box (mm).
   - `struct DesignDomain { VoxelGrid grid; DesignMask mask; int offset_i/j/k; }`.
   - `DesignDomain expand_design_domain(const VoxelGrid& part, const DesignBox&
     design_box, const std::vector<DesignBox>& keep_out = {})` — builds the larger
     grid (same spacing/lattice as the part, spanning the union of the part bbox
     and the design box) and the effective mask: imported-part solid voxels →
     **FrozenSolid** (never removed), keep-out voxels → **FrozenVoid** (tagged
     Empty so they carry no FEA element / no self-weight), the rest of the design
     volume → **Active** (tagged Interior — new growable material), everything else
     → Empty. Box membership is decided by the voxel CENTRE (matches the
     voxelizer's centre-sampling).
   - `int remap_node_to_domain(const VoxelGrid& part, const DesignDomain&, int
     node)` — maps a part corner-node id to the expanded grid (shifted by the
     offset). Used to remap BCs / external loads.

2. **`core/src/voxel/voxelize.cpp`** — implements the three above. This is an
   ALWAYS-built (Eigen-free) TU, so the corner-node index is computed with the
   inlined `fea.hpp` formula rather than calling `fea_node_index` (which lives in
   the Eigen-gated `assembly.cpp`) — no new link dependency. Added `<string>`.

3. **`core/include/topopt/pipeline.hpp`** — `MinimizePlasticOptions` gains
   `std::optional<DesignBox> design_box` (added `<optional>`) and
   `std::vector<DesignBox> keep_out_boxes`. Both default to unset/empty. Fully
   documented, including the byte-identity guarantee when unset.

4. **`core/src/simp/minimize_plastic.cpp`** — when `design_box` is set the driver
   expands onto the larger grid, remaps `bcs` and `external_loads` onto it, builds
   the effective mask from `expand_design_domain`, and runs the WHOLE existing
   pipeline (mask-aware MMA/SIMP, stress recovery, V3 suite, viz fields, ladder)
   on the expanded grid via local `const VoxelGrid& G` / `const
   std::vector<DirichletBC>& B`. When unset, `G`/`B` alias the caller's
   `grid`/`bcs` and every derived quantity is byte-for-byte identical to the
   pre-M7.dom-core driver (the branch is simply skipped). A caller `design_mask`
   together with a `design_box` throws (the box builds the mask itself).

5. **`core/tests/validation/test_design_domain.cpp`** (new) + **CMakeLists.txt**
   target `design_domain` (Eigen-gated, `SETTINGS_RULES_PATH` injected).

**No changes** to the FEA, the density filter, the MMA updater, the stress path,
the infill knockdown, the anchor-integrity fixes, or the ladder logic. Nothing in
`/app/`, `materials.json`, or `ROADMAP.md`.

## Acceptance test (the proof the feature works)

`test_design_domain.cpp` builds a THIN L-bracket (8×3×8 voxels, arms 2 thick) in
code — mounted at the top of the vertical arm, loaded downward at the free end of
the horizontal foot so the re-entrant INNER corner is the stress concentration —
and wraps it in a corner design box (extended 1 voxel below the origin in x and z
to force NONZERO node-remap offsets, and 2 voxels beyond the far faces), with a
keep-out box on the outer notch corner. Asserts (20 checks, all pass):

- **(a)** the optimizer GROWS material in the Active design volume (added solid
  voxels where the import had none): **133** Active voxels became solid.
- **(b)** the keep-out region stays empty (every FrozenVoid voxel density == 0).
- **(c)** the imported part is NEVER removed (min FrozenSolid density ≥ 0.9; the
  §7 V3 Load/Fixture retention gate holds).
- **(d)** the grown result has LOWER compliance than the import alone:
  `import_only = 9.24e-10` → `grown = 5.11e-11`, **94.5 % lower** — adding the
  gusset along the load path stiffened the bracket.
- Expansion enlarges the grid (84 solid → 363 voxels; frozen 84 / void 12 /
  active 267), the part sits at the expected offset (1,0,1), node remapping is
  exercised end-to-end (a wrong remap would misplace the clamp and wreck the
  solve), and the DEFAULT (no design box) run stays on the imported grid.

## Default behavior is byte-identical

With `design_box` unset the design-box branch is skipped and `G`/`B` alias the
caller's inputs, so the mask (`make_active_mask(grid)` or the caller mask), the
loads, and every downstream computation are bit-for-bit the pre-M7.dom-core
driver. The existing `minimize_plastic` and `passive_regions` gates exercise that
path unchanged. The acceptance test also asserts an unset run is on the imported
grid (not an expanded one).

## Notes / follow-ups

- The self-weight path under a design box (empty `external_loads`) computes
  self-weight over the expanded grid's solid voxels (frozen part + Active design
  envelope), held fixed across rungs — a documented modeling choice mirroring the
  existing original-solid self-weight model. The acceptance test drives the
  primary external-load path; the self-weight+box default is documented but only
  lightly exercised (its bcs-remap machinery is shared and covered).
- The UI to size the design box is the SEPARATE, GATED **M7.dom-app** task — not
  built here (scope guard: pure core).
