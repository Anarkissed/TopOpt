Public API — `core/include/topopt/fea.hpp` (extended; still **Eigen-free**):

  Mises field (size `grid.voxel_count()`, indexed like the grid), each solid
  voxel evaluated at its element centroid from a displacement solution; empty
  voxels are 0. This is the "von Mises stress field output" the task requires.

### How the gate is set up and why it is trustworthy
Four beam cases, each run at three mesh resolutions (elements through the beam
depth `m = 4, 8, 12`), deflection compared to Euler–Bernoulli:
- **cantilever** (clamp the whole x=0 face): tip **point** load δ=PL³/(3EI), and
  uniformly **distributed** load δ=WL³/(8EI); deflection = mean uₓ over the tip
  plane.
- **simply-supported** (vertical uₓ supports on the bottom edge at each end +
  minimal in-plane pin/roller): midspan **point** load δ=PL³/(48EI), and
  **distributed** load δ=5WL³/(384EI); deflection = mean uₓ over the midspan
  plane.

Key physics that makes ≤2% vs Euler–Bernoulli both true and non-trivial:
1. **Locking → convergence from below.** Fully-integrated Hex8 elements shear-
   lock in bending, so a coarse mesh is too *stiff*; h-refinement releases the
   locking and the deflection converges upward toward the true 3D answer. The
   three meshes show this (coarsest is outside 2%, finer two inside).
2. **Slenderness for E–B accuracy.** Euler–Bernoulli neglects transverse shear;
   its error vs the true 3D answer scales as (H/L)² and is *larger* for the S-S
   cases (the S-S point case's shear correction is ~2.2% at L/H=12 — Euler–
   Bernoulli itself is then out of tolerance and the FEA converges to +2.6%,
   confirmed empirically). I use **L/H = 20** uniformly, which keeps the shear
   correction well under 2% for every case.
3. **Scale-invariance.** Linear-elastic relative error depends only on the
   dimensionless shape (L/H, B/H, ν) and `m`, not on absolute size, E, or load
   magnitude — so scaling the integer grid by `m` at fixed spacing *is* mesh
   refinement of one fixed physical beam.

Per-case assertions (tolerance = the V1 2% gate, `kGate = 0.02`):
- finest mesh error ≤ 2%, **and** second mesh error ≤ 2% (sufficient resolution
  reached and maintained);
- `|e_coarsest| > |e_finest|` (refinement reduces error);
- `|e₃−e₂| < |e₂−e₁|` (successive-mesh change shrinks → convergence; sign-robust,
  needed because the S-S error crosses zero as locking is released).

Measured relative errors (deterministic — CG tol 1e-10, so discretization, not

## What I did NOT do
- **Did not evaluate stress anywhere but the element centroid.** `hex8_stress`
  accepts (ξ,η,ζ) but `fea_von_mises_field` samples only the centroid (0,0,0),
  the standard single representative point for a trilinear hex. No Gauss-point
  sampling, nodal extrapolation, or inter-element stress averaging/recovery
  (e.g. superconvergent patch recovery) — not needed for M2.4 and §8 forbids
  speculative abstraction. Centroid stress under-reads the peak surface fiber
  stress, so the field is for relative comparison, not a certified peak stress.
- **Did not assert a tight bending-stress magnitude on the beams.** The beam von
  Mises sanity only checks finiteness, positivity, and that the peak sits in the
  root half. Centroid-sampled bending stress from a locking element is a poor
  absolute match to Mc/I, so a tight magnitude assertion would be fragile. The
  *exact* von Mises correctness is carried by the uniaxial patch, where the
  answer is analytic.
- **Did not use different slenderness per beam.** I use L/H = 20 for all four
  cases even though the cantilever passes comfortably at L/H = 12; a single
  slenderness is simpler to defend and the S-S point case needs 20.
- **Did not model true point loads or knife-edge supports.** Loads and S-S
  supports are spread over a face/edge of nodes (Saint-Venant: local detail
  doesn't affect the slender-beam deflection). A literal single-node point load
  or single-node support would inject a local singularity.
- **Did not add reaction-force output, principal stresses, or a stress tensor
  field type.** `fea_von_mises_field` returns the scalar field only. Full-tensor
  field output can be added when a consumer needs it (M3/M5 reports).
- **Did not touch the `fea_solve_cg` singularity behaviour.** The S-S BCs fully
  remove rigid-body motion (verified: no throw, `converged`), so the M2.3 caveat
  (CG doesn't guard consistent-singular systems) is not exercised here. The M3.1
  void-DOF safety gate is still the place that must handle under-constrained SIMP
  systems.
- Did not touch `/app/`, `ARCHITECTURE.md`, `materials.json`, CI config, any
  fixture, or any existing test/assertion/tolerance. The ROADMAP change is the
  single M2.4 box.

## Warnings for the next run
- **The V1 result is a property of Euler–Bernoulli's slenderness limit, not just
  the FEA.** If a future change re-runs these beams at a smaller L/H, the S-S
  point case will legitimately exceed 2% because Euler–Bernoulli itself becomes
  inaccurate — that is not an FEA regression. The honesty check above documents
  this explicitly. Do not "fix" such a failure by loosening the 2% tolerance
  (rule 1) — increase slenderness instead.
- **`beam_validation` takes ~60 s** (the two L/H=20, m=12 S-S solves are ~4000
  CG iterations at ~122 k DOFs each). Full suite ~87 s locally, well under the
  M2.3 "< 5 min CI budget". If someone raises `m` or L/H, watch the wall time.
- **CG cap is 20000, tolerance 1e-10.** Observed max ~4000 iterations (S-S,
  m=12), so ~5× headroom; the iteration count is deterministic per matrix but
  may drift a few percent on CI's vcpkg Eigen vs local homebrew — the cap
  absorbs that. The *deflection errors* are set by discretization (solver tol is
  1e-10), so they are stable across platforms to well within the assertion
  margins.
- **`fea_von_mises_field` reads displacements only via `FeaSolution::at`** and
  re-derives the element material — it does not know which E/ν produced `sol`.
  Pass the same material you solved with, or the stresses are wrong (they scale
  with E). For SIMP (M3) the per-voxel modulus is `E·ρ^p`, so a uniform-E field
  call will be wrong on a graded density field — M3 will need a per-voxel-E
  stress path.
- **Node / DOF / Voigt conventions still load-bearing** and identical to
  M2.1/M2.2/M2.3. `hex8_stress` reuses them; a mismatch would silently corrupt
  stress recovery. The uniaxial-patch and pure-shear analytic checks are the
  guard.
