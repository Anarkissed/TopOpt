## What I did
Pure core (`/core/`); no `/app/`, no fixtures modified, no dependency added.

- **`core/include/topopt/simp.hpp`**: added the public stress-constrained API and
  its formulation record (P-norm aggregation, qp relaxation, adjoint):
  - `StressAggregate` + `simp_stress_aggregate(...)` — the aggregated relaxed von
    Mises measure `g = (sum_e sigma~_e^P)^(1/P)` for a physical density field and
    its exact adjoint sensitivity `dg/drho`, plus `max_von_mises` (true peak over
    printed voxels) and `max_relaxed`. One primal + one adjoint CG solve.
  - `StressConstraint` (stress_cap, p_norm, relaxation_q, printed_threshold) +
    `simp_optimize_stress(...)` — minimum compliance with a volume constraint AND
    the aggregated stress constraint, on the MMA path.
- **`core/src/simp/simp.cpp`**: implemented the above.
  - The relaxed stress `sigma~_e = clamp(rho_e)^q * sigma_vm(D0 * B * u_e)` — von
    Mises of the SOLID-material centroid stress (constitutive `D0` at E0/nu),
    relaxed by a density power `q < p` (qp relaxation, Bruggi 2008) so a void
    voxel's constraint switches off (the stress singularity problem). `q = 0.5`.
  - Aggregation is the **P-norm** with **P = 8** (an upper bound on the true max,
    approaching it from above). Recorded in code + below for DECISIONS.md.
  - **Adjoint** `dg/drho`: explicit part (the `rho^q` factor) + implicit part
    through `u` via ONE adjoint solve `K(rho) lambda = dg/du` against the same
    penalized stiffness, reusing the void-gated `fea_solve_cg` (K symmetric, so
    the adjoint operator is K; the RHS is passed as the "load" vector). The
    centroid stress-displacement matrix `DB = D0*B` is built once (all cubic
    voxels share it).
  - **Two-constraint MMA** (`mma_update_stress`): the Svanberg moving-asymptote
    machinery of M7.mma.1 extended to m = 2 (volume + stress). The dual is
    concave in lambda >= 0; its stationarity is solved by nested bisection (inner
    lambda1 holds the volume constraint, outer lambda2 holds the stress
    constraint), with KKT complementarity (a satisfied constraint keeps its
    multiplier at 0). The single-constraint `mma_update` (M7.mma.1) is untouched.
  - Adaptive normalization (Le/Paris/Kim/Tortorelli 2010) scales the P-norm
    toward the true peak of the relaxed field each iteration, held constant
    across a subproblem so the recorded sensitivity stays exact; plus a
    cap-continuation ramp (tighten the effective cap ~8%/iteration toward the
    target) so an aggressive cap does not oscillate.
  - Scope guards: `simp_optimize_stress` uses MMA regardless of `options.updater`
    and rejects a Heaviside projection schedule (as the plain MMA path,
    M7.mma.1), plus `stress_cap <= 0`, `p_norm <= 1`, `relaxation_q` not in
    `(0, penalty)`.
- **`core/tests/validation/test_stress.cpp`** (new) + **`core/CMakeLists.txt`**
  (register `test_stress`, Eigen-gated; when OCCT is present it also consumes the
  committed L-bracket fixture — `TOPOPT_HAVE_STEP` + injected `DEMO_FIXTURE_DIR`).

## Design choices & why
- **The adjoint is FD-verified.** `test_stress` part 1 perturbs each physical
  density by ±h and recomputes the P-norm aggregate (each a full primal solve),
  matching the analytic `dg/drho` (one adjoint solve) to worst relative error
  **2.1e-7** on a tiny grid — the M7.mma.2 sensitivity gate.
- **Load-case calibration is verified against the fixture's independent
  reference.** With the fixture load case (anchor = top of the upright wall
  z = 60, wall x in [22,30]; load = 1220 N down over the far plate face x = −30,
  z in [0,8]) my full-solid peak von Mises is **118.3 MPa at res 24**, matching
  the fixture's independent NumPy reference (9.81e-2 MPa/N × 1220 N = 119.7 MPa)
  to < 2%. So the geometry/BC/load wiring reproduces the maintainer's reference.

## Warnings for the next run
- **`simp_optimize_stress` correctly enforces a REACHABLE cap** (the feasible
  cantilever gate drives the aggregate to the cap within tol and compliance rises)
  but plain MMA plateaus ABOVE an aggressive cap that sits below the achievable
  stress floor (see Blocked). If a later task tightens caps, expect to need
  GCMMA-style conservative approximations or fine-mesh continuation.
- **The relaxation can be "gamed" at high q.** With `relaxation_q` near the
  penalty (e.g. 2.5), the optimizer satisfies the RELAXED aggregate by parking
  high-stress material at intermediate density (rho ~ 0.5) where `rho^q` masks it,
  while the TRUE von Mises there stays high and the structure disconnects
  (compliance blows up). Keep `q = 0.5` (relaxed ~= true on solid material); do
  not raise it to force the aggregate down.
- **Cap continuation + adaptive normalization run every iteration.** They are
  cheap (no extra solve) but assume the first iteration seeds them; if a future
  change resets the loop mid-run, re-seed `cap_eff`/`c_norm`.
- **`test_stress` runs two ~100-iteration optimizations plus an FD sweep (~12 s
  locally); the L-bracket section adds one full-solid solve at res 16.** It is
  Eigen-gated (FD + feasible case) with the L-bracket section additionally
  OCCT-gated (`TOPOPT_HAVE_STEP`).

Findings (measured; the implementation and setup are verified correct — the
adjoint FD-checks to 2e-7 and the full-solid peak matches the fixture's NumPy
reference to < 2%):

1. **The true peak von Mises can never reach the cap — at ANY resolution.** The
   minimum-stress design is the full-solid part (adding material lowers stress),
   and its peak already exceeds the cap: **84 MPa (res 16), 118 MPa (res 24),
   151 MPa (res 32), 165 MPa (res 48, the fixture's own number)** vs the 55 MPa
   cap. The gap GROWS with resolution (the re-entrant corner is a singularity).
   This is exactly the fixture's `unconstrained_exceeds_cap` note ("the singular
   corner guarantees this at any resolution"). So invariant (b) cannot concern the
   raw peak; it must concern the qp-RELAXED aggregate, satisfied by voiding the
   corner into a stress-relieving fillet.
2. **A stress-relieving fillet needs a fine mesh.** At res ≤ 24 the corner is 1–2
   voxels; no fillet forms, so the relaxed-aggregate floor stays well above the
   cap (honest q = 0.5: ~166 MPa at res 16). A smooth fillet that brings the
   aggregate to ~55 MPa only forms around the fixture's res 48.
3. **Res 48 is not CI-affordable here.** `cli_demo` (res 48, one ladder) already
   takes ~460 s; a stress test needs an unconstrained + a constrained run (the
   constrained one does two solves/iteration), i.e. ~5–10 min for one test, and
   at that resolution plain MMA's non-conservative convex model of the highly
   non-linear stress constraint is not guaranteed to converge to the cap.
