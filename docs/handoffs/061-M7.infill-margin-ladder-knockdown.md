# Handoff 061 — M7.infill-margin: infill-derived acceptance-gate knockdown

## Task
M7.infill-margin (named assignment, per DECISIONS 2026-07-11 infill handling):
apply an infill-derived **knockdown** to the worst-case stress margin used by the
reduction-ladder **acceptance** test, so the optimizer stops stripping material
against a solid-part margin the real sparse print won't deliver. **Infill does NOT
enter the FEA** — no change to element stiffness, the stress field, or the solver
(ARCHITECTURE §2 stands). This is a scalar applied to the margin **at the
acceptance gate only**. TRACK core (`/core/` + the bridge threading of the infill
value in `TopOptBridge/bridge.cpp`, strictly as needed).

## The knockdown curve (SEED — maintainer tunes; NOT final)
```
f = infill_percent / 100
knockdown(f) = 1.0               for f >= 1   (solid/unset — returned exactly, no arithmetic)
             = max(f^1.5, 1e-3)  for 0 < f < 1
             = 1e-3              for f <= 0   (degenerate hollow floor)
```
Constants `kKnockdownExponent = 1.5` and `kKnockdownFloor = 1e-3` are named at the
top of the helper.

**Reasoning.** Gibson-Ashby cellular-solid scaling: the effective/solid strength of
a porous (infilled) region scales roughly as `(relative density)^p`. Using infill
fraction `f` as the density proxy and the foam strength exponent range `p ≈ 1.5–2`,
I seed `p = 1.5`. Properties of this choice:
- `p > 1` puts the curve **below** the linear `f` for `f ∈ (0,1)` (sub-linear) and
  pins it to **1.0 at `f = 1`** (solid) — the shape the task asked for.
- It **ignores the load real perimeters/walls carry** (a slice always prints solid
  outer walls), so it **under-estimates** true part strength → **conservative**:
  it knocks the margin down harder than reality, stopping the ladder sooner and
  retaining more material. Safe direction for a seed.
- Range stays strictly in `(0,1]` (floor keeps it positive even at `f → 0`).

**Tuning levers** for whoever calibrates against real prints: raise `p` toward 2
for a less-aggressive knockdown; add a wall-count-derived floor (walls carry load
the pure-infill model omits, so real strength has a higher floor than `f^p`);
or replace the power law with a measured strength-vs-infill relation. Isolated in
one helper for exactly this.

## Back-compat (asserted, not assumed)
- `knockdown(100)` returns literal `1.0` with **no arithmetic**, and
  `margin.worst_case * 1.0 == margin.worst_case` bit-for-bit in IEEE-754 (holds for
  finite and ±inf; margins can be +inf when a stress is 0). So the gate is
  byte-identical to the pre-M7.infill gate when infill is unset.
- New **Scenario M** directly asserts unset-infill vs explicit-100% produce a
  **byte-identical report JSON**, and that the solid part still accepts the whole
  ladder (unchanged accepted-rung selection).
- All 29 ctest targets pass unchanged (see evidence) — including `minimize_plastic`,
  the determinism gate, and the full `cli_demo` end-to-end (which drives the driver
  with default options).

## Tests (tests-first) — `core/tests/validation/test_minimize_plastic.cpp`, Scenario M
Gravity is placed analytically from Scenario A's calibrated per-rung margins (which
scale as `1/gravity`) so the SOLID part comfortably accepts the WHOLE ladder — the
"raw margin is large" over-stripping case the knockdown targets (weakest rung's
solid margin set to 4× the 1.5 threshold). Assertions (curve-agnostic, so maintainer
re-tuning of `p`/floor won't break them — they test behavior, not the exact factor):
1. Unset infill == explicit 100% infill (byte-identical report); solid accepts all
   3 rungs, no margin stop.
2. **A LOW infill (30%) accepts strictly fewer rungs and stops on margin** — the
   over-stripping is demonstrably reduced. Observed: `solid_accepted=3
   low(30%)_accepted=2 stopped=1`.
3. Accepted-rung count is **monotone non-increasing** across an infill sweep
   {100,70,50,30,10}.
4. Every rung the knockdown accepts still clears the **raw** solid margin (the gate
   only ever rejects MORE; it never accepts something the solid gate would reject).
5. Every rung the low-infill run RETAINED has stored margin/stress/volume-fraction
   **identical** to the solid run (proves infill touches only the accept/reject
   decision — not the FEA, the stress field, or the optimized geometry).
