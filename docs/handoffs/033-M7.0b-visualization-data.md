### API — `core/include/topopt/pipeline.hpp`
`MinimizePlasticVariant` gains four members (documented inline):
- **(a)** `std::vector<double> von_mises_field` — per-voxel von Mises over the
  printed material, grid-indexed (size `grid.voxel_count()`), MPa. Nonzero only
  on printed voxels (physical density > 0.5, the M3.5 iso the mesh is extracted
  at); zero on void/Empty. This is the value the peak-stress reduction already
  computes per element, now retained per voxel.
- **(c)** `int support_volume_voxels` — the M4.3 `support_overhang_voxels` proxy
  for the analysed build direction (`report.orientation`), evaluated over THIS
  variant's PRINTED geometry (density > 0.5), i.e. the per-variant printed shape,
  not the fixed original solid grid.
- **(d)** `double mass_grams` — density (g/cm³) × printed volume, spacing-aware:
  `(# printed voxels) * grid.voxel_volume() (mm³) / 1000`.
- **(b)** `const TriangleMesh& mesh() const { return v3.mesh; }` — a zero-copy
  accessor exposing the isosurface `check_v3` already extracted into `v3.mesh`.
  No marching-cubes recompute, no mesh copy. Empty for a cancelled rung.

The `MinimizePlasticResult::cancelled` doc comment was extended to name the four
new fields as also default-constructed on the cancelled rung.

### Driver — `core/src/simp/minimize_plastic.cpp`
In the existing per-rung stress loop (after the cancelled-rung early-out):
- (a)+(d): the loop that recovers element stresses now also assigns
  `variant.von_mises_field[idx] = st.von_mises` for each printed voxel and counts
  printed voxels; `mass_grams` is computed from that count, `grid.voxel_volume()`
  and `material.density_g_cm3`, dividing by 1000 to convert the mm-MPa g/cm³
  product to grams.
- (c): a copy of the grid with non-printed voxels retagged `Empty` is passed to
  `support_overhang_voxels(printed_grid, build_dir)`. The proxy is defined on
  grid tags (solid = non-Empty), so masking the carved voxels to Empty makes it
  operate on the variant's printed shape.
- (b): nothing added — `mesh()` returns the `v3.mesh` the existing
  `variant.v3 = check_v3(grid, rho, kIso)` already produced.

`build_dir` (already computed as the unit negation of the gravity direction) is
the "analysed build direction" for (c). `<atomic>`/`<functional>` etc. unchanged;
`orient.hpp` (support proxy) and `voxel.hpp` (`VoxelTag`) were already included.

  stored once in `v3`; a second copy would be waste and drift risk. The M7.1
  Swift bridge reads `variant.mesh()` (or `variant.v3.mesh`); both are the same
  object.
- **Did NOT store the full per-voxel stress TENSOR** (the six-component `sigma`),
  only the scalar von Mises the task names. The stress overlay (M7.8) uses von
  Mises → vertex colors; the tensor stays a local in the driver.
- **Did NOT expose a printed-voxel COUNT or a cm³ volume field.** `mass_grams`
  and `support_volume_voxels` are the two the task lists; the app converts the
  support count to cm³ via `grid.voxel_volume()` at display time (M7.8), and the
  printed volume is recoverable as `mass_grams / density * 1000` or by counting
  `von_mises_field`/density > 0.5.
- **Did NOT change what "printed" means.** A voxel is printed iff physical
  density > 0.5 (the M3.5 iso `kIso`) — the same threshold the variant mesh, the

## Warnings for the next run
- **`support_volume_voxels` is over the PRINTED subgrid, not the original solid
  grid** — it therefore VARIES across variants (a lighter variant carves more
  material, generally raising overhang count). If a future consumer wants the
  support of the un-optimized part, that is a different quantity and not this
  field. The M7.8 "support estimate (cm³)" = `support_volume_voxels *
  grid.voxel_volume()` (mm³) / 1000.
- **`mesh()` returns a reference into `v3.mesh`.** It is valid only as long as
  the `MinimizePlasticVariant` (and thus its `v3`) lives. The M7.1 bridge must
  copy vertices/indices across the Swift boundary, not hold the reference past
  the result's lifetime.
- **The von Mises field is grid-indexed and zero off the printed material**, so
  it is NOT a dense field over the whole part — a viewer coloring the mesh must
  sample it at the printed voxels the mesh encloses (density > 0.5), where it is
  meaningful. Empty/void entries are 0 by construction, not "zero stress".
- **`mass_grams` uses the mm-MPa-g/cm³ unit fold** (÷1000 for cm³→mm³). It is
  correct only when the grid spacing is in mm (the whole pipeline's assumption,
  documented on `MinimizePlasticOptions.gravity` and in run_job). A grid built
  in other length units would need the conversion revisited.
- **Scenario I runs vf = 1.0** (the whole part kept) — a legitimate ladder value
  (the validator allows (0, 1]) chosen so "printed == solid" is exact and the
  mass is hand-checkable. It is not a typical minimize_plastic ladder rung.
- **The stress solve is unchanged** — retaining the per-voxel von Mises adds one
  `vector<double>` assign + one store per printed voxel, negligible next to the
  CG solve; `support_overhang_voxels` on the printed subgrid is one O(voxels)
  pass. No measurable per-rung cost change.
