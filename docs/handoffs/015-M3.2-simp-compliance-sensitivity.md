## What I did
Added a SIMP module (`simp/` per ARCHITECTURE §3) that realises the classic
Sigmund 99-line material law and the compliance objective + gradient, and added
the per-voxel-modulus assembly primitive it needs to the FEA module — routed
through the M3.1 void-DOF gate as the ROADMAP requires ("SIMP FEA solves should
call `fea_solve_cg`").

Public API — `core/include/topopt/simp.hpp` (new; Eigen-free, like fea.hpp):
- `struct SimpParams { youngs_modulus (E0), poisson, penalty=3, density_min=1e-3 }`.
- `double simp_youngs(params, density)` — the interpolation law
  `E(rho) = clamp(rho, rho_min, 1)^p * E0`.
- `std::vector<double> simp_uniform_density(grid, value)` — the initial design
  field (solids = value, Empty = 0), size `grid.voxel_count()`.
- `struct SimpCompliance { FeaSolution solution; double compliance;
  std::vector<double> dcompliance; CgInfo cg; }`.
- `SimpCompliance simp_compliance(grid, params, density, bcs, loads, tol, maxit)`
  — the penalized solve + compliance `c = f^T u = sum_e E(rho_e) u_e^T K_unit u_e`
  + per-voxel sensitivity `dc/drho_e = -p rho_e^(p-1) E0 u_e^T K_unit u_e` (0 on
  Empty voxels).

Public API — `core/include/topopt/fea.hpp` (extended; still Eigen-free): a graded
overload
`fea_solve_cg(grid, const std::vector<double>& youngs_per_voxel, poisson, bcs,
loads, tol, maxit, info)`. Each solid voxel uses its own Young's modulus; since
the isotropic Hex8 stiffness is exactly linear in E, the element is
`E_voxel * K_unit`. This is the "SIMP-penalized stiffness" assembly and goes
through the same void gate + Jacobi-CG solve as the uniform overload. WHY a new
FEA function rather than assembling inside `simp.cpp`: reusing the delicate M3.1
void-gate + BC-reduction (handoff 014 warns it is fragile) beats a second copy,
and per-voxel modulus is a legitimate FEA capability; `simp/` keeps only the
penalization math and sensitivities.

### Material-law choice and interaction with the M3.1 gate
Classic 99-line SIMP `E(rho)=rho^p E0` with a density floor `rho_min` (not the
modified `E_min + rho^p(E0-E_min)`), matching ARCHITECTURE §4's "Standard Sigmund
99-line formulation". With `p=3`, `rho_min=1e-3` a floor voxel has stiffness
`~1e-9 * E0` — relative diagonal `~1e-9`, above the M3.1 gate's `1e-12` filter
threshold, so SIMP design voxels are never dropped by the void gate; only
genuinely Empty (out-of-design) voxels are.

### Why the sensitivity FD check is trustworthy
`test_simp` section 4 (the M3.2 gate): a 4×2×2 cantilever with a deterministic
non-uniform density in [0.4, 0.9] (strictly inside `(rho_min,1)` so the clamp is
inactive and `E(rho)` is smooth). For 5 probe voxels it perturbs `rho_e` by
`±1e-4`, re-solves (CG tol `1e-13`), and compares the central difference
`(c(+h)-c(-h))/(2h)` to the analytic `dc/drho_e`. Worst relative error observed
**7.10e-08** against a `1e-3` gate — a wrong exponent (`p` vs `p-1`) blows this to
150% (verified below). Correctness of the compliance itself is anchored two
independent ways in section 3: a uniform density field reproduces the
uniform-material `fea_solve_cg(E=rho^p E0)` DOF-for-DOF, and the returned
`compliance` equals `f^T u` recomputed straight from the loads and solution.

## What I did NOT do
- **Did not add the density filter or the Optimality-Criteria update** — those are
  M3.3. `simp_compliance` returns the raw (unfiltered) sensitivity field.
- **Did not add a volume constraint, volume-fraction target, or any optimizer
  loop** — that is M3.4. This task delivers only the objective and its gradient.
- **Did not implement modified SIMP** (`E_min + rho^p(E0-E_min)`); I used the
  classic 99-line `rho^p E0` with a `rho_min` floor per ARCHITECTURE §4. A design
  voxel can therefore never reach exactly zero stiffness (floor `rho_min^p E0`).
- **Did not apply the sensitivity clamp at the density bounds.** `dc/drho_e` is
  the analytic derivative of `clamp(rho,rho_min,1)^p`; at a clamped voxel
  (`rho<=rho_min` or `rho>=1`) `simp_compliance` still evaluates it at the clamped
  `rho`, so the reported derivative uses the boundary value (the sub-gradient is
  the caller's concern for M3.3's OC update). All FD-tested voxels are strictly
  interior.
- **Did not route the graded solve through the direct solver `fea_solve`.** Only
  `fea_solve_cg` gained a graded overload (the void-gated path M3.1 mandates for
  SIMP). `fea_solve` is untouched.
- **Did not add per-voxel stress recovery for graded density.** `fea_von_mises_
  field` still assumes a single uniform E; a SIMP-graded stress field (per-voxel
  `E(rho)`) is a later concern (M2.4 handoff flagged this).
- **Did not evaluate `q_e = u_e^T K_unit u_e` at anything but the full element
  matrix** — it is the exact element strain energy per unit modulus (consistent
  with the assembled compliance), not a Gauss-point sample.
- Did not touch `/app/`, `ARCHITECTURE.md`, `materials.json`, CI config, any
  fixture, any existing test/assertion/tolerance, or any header beyond adding the
  graded `fea_solve_cg` declaration. The ROADMAP change is the single M3.2 box.

## Warnings for the next run
- **CI required before merge (DECISIONS.md 2026-07-03).** Local green is not
  sufficient; open a PR and confirm the Linux vcpkg pipeline
  (`-DTOPOPT_REQUIRE_DEPS=ON`) is green, then paste the Actions URL into the Test
  evidence section. The CG iteration counts / FD errors are set by discretization
  (CG tol `1e-13`), so they are stable across CI's vcpkg Eigen vs local homebrew;
  the FD relative error (`7e-08`) has ~4 orders of margin under the `1e-3` gate.
- **The graded `fea_solve_cg` shares `solve_reduced_cg` with the uniform one.** A
  bug in that helper now breaks both overloads (and thus M2.3, M2.4, M3.2). The
  `fea_cg`/`beam_validation` tests guard the uniform path; `simp` guards the graded
  path.
- **`simp_compliance`'s sensitivity is the RAW compliance gradient**, not filtered.
  M3.3 must apply the density filter to the sensitivities (or densities) before
  the OC update, or the optimizer will produce checkerboarding — the classic
  99-line failure mode. Do not mistake the raw `dcompliance` for a filtered one.
- **`density` is grid-indexed (size `voxel_count()`), not solid-packed.** Entries
  for Empty voxels are ignored on input and their `dcompliance` is 0 on output.
  Feed the SAME material to `simp_compliance` you intend to optimize; the compliance
  scales with `E0` and `rho^p`.
- **Node / DOF / Voigt conventions remain load-bearing** and identical to
  M2.1–M3.1. `simp_compliance` reuses `fea_element_nodes` and `hex8_stiffness`;
  the uniform-match and FD checks are the guard against a convention mismatch.
