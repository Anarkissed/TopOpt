## What I did
Implemented a STEP importer over OCCT and wired a new `step_import` ctest.

New public header (the API the app/plugins see, ARCHITECTURE §3 — OCCT stays out
of it):
- `core/include/topopt/step.hpp`: `StepError`, `StepTessellation`
  (`linear_deflection` = the "controllable deflection" the task calls for, plus
  `angular_deflection`), `StepModel` (tessellated `TriangleMesh` from M1.3's
  `mesh.hpp`, plus exact-B-rep `brep_volume`, `solid_count`, `face_count`), and
  `import_step_file(path, tess)`. STEP I/O lives in `io/` per §3; it reuses the
  M1.3 `mesh/` types (`signed_volume`, `check_watertight`) rather than a parallel
  mesh type, matching the §5 pipeline "tessellated surface → watertight check".

Implementation:
- `core/src/io/step.cpp`: probe the path first for a clear missing-file
  diagnostic, then `STEPControl_Reader` (handles AP203/214/242; the fixtures are
  AP242), `TransferRoots()`/`OneShape()`. Exact measures come from the B-rep and
  are independent of tessellation: `solid_count`/`face_count` via
  `TopExp_Explorer`, `brep_volume` via `BRepGProp::VolumeProperties`. Tessellation
  is `BRepMesh_IncrementalMesh` at the requested linear/angular deflection; each
  face's `Poly_Triangulation` is transformed by its `TopLoc_Location` and welded
  into one `TriangleMesh`. Triangle winding is flipped for `TopAbs_REVERSED` faces
  so all normals face outward (the OCCT-standard idiom its own STL/glTF writers
  use), which makes the divergence-theorem `signed_volume` positive. Vertices are
  merged with a quantized weld (1e-6 mm grid — far below any deflection, far above
  numeric noise) so shared-edge and periodic-seam duplicate nodes collapse into a
  closed 2-manifold for `check_watertight`.

Tests — `core/tests/unit/test_step.cpp`, registered as ctest `step_import`
(same self-contained CHECK harness as `test_stl.cpp`; no test framework, §4).
`STEP_FIXTURE_DIR` is injected by CMake (same pattern as `STL_FIXTURE_DIR`).
Assertions cite the human-authored `expected_values.json` as literals.

Build:
- `core/CMakeLists.txt`: when OCCT is found, compile `src/io/step.cpp` into
  `topopt` and link the OCCT toolkits it needs (STEP read + mesh + gprop +
  topology), with the OCCT include dir marked `SYSTEM`; and build the `test_step`
  executable + `add_test(step_import)`. Under `TOPOPT_REQUIRE_DEPS=ON` (CI) OCCT
  is `REQUIRED`, so `step_import` always runs in CI. Under `OFF`, OCCT is looked
  up `QUIET`: if absent, `step.cpp`/`test_step` are skipped so the OCCT-free
  targets (materials, STL) still build for a dependency-light local checkout.
- The cube fixture rename `10 mm cube.step` → `cube.step` was committed
  separately (`43f94d4`) as a byte-identical, maintainer-directed change; the
  agent authored no fixture content (see "What I did NOT do").

- cube.step: 1 solid, 6 B-rep faces; `brep_volume` == 1000 mm³ (rel 1e-6);
  tessellates to 12 triangles / 8 welded vertices, watertight (0 boundary / 0

  coarse (0.1) and fine (0.001) deflection — proving a planar solid is exact at
  any deflection; bbox min (-5, -5.384615385, 0) / max (5, 4.615384615, 10)
  (the fixture's non-origin-centred placement).

  it, the finest is within 0.1% (rel 1e-3), and the coarsest is NOT yet within
  0.1% (so the convergence assertion is non-vacuous). Measured: 1555.55 →
  1560.72 → 1570.38 vs analytic 1570.796 (reldiff 9.7e-3 → 6.4e-3 → 2.6e-4).

## Test-honesty checks (throwaway builds in scratchpad, nothing committed)
- Break the vertex weld (`kEps` 1e-6 → 1e-30 so nothing merges): exactly the
  watertight + volume-convergence checks fail (3/30). Confirms the weld is what
  closes the mesh.
- Flip ALL faces (`reversed = true` always): cube/cylinder mesh volumes invert
  to −1000 / −1570.38 — proving `signed_volume`'s sign is genuinely exercised
  and the outward-winding logic drives it.
- Both `cube.step` and `cylinder.step` sit with a face on the z=0 plane, whose
  triangles contribute exactly zero to the divergence-theorem volume; the single
  REVERSED face in each is that z=0 face, so flip-for-REVERSED vs no-flip give
  identical results HERE (verified: both +1000/+1570.38). The flip is still the
  correct general idiom (see flip-ALL above); these two fixtures just can't
  distinguish it. Documented so the next run doesn't "simplify" the flip away.

  warnings are isolated via a SYSTEM include).

## Warnings for the next run
- **expected_values.json key vs filename mismatch (cosmetic, maintainer to
  reconcile if desired):** the cube's entry is keyed `"10_mm_cube.step"` while
  the file is now `cube.step`. `test_step.cpp` loads `cube.step` and asserts the
  entry's literal values, so this is functionally harmless, but the key no longer
  matches any filename. Fixtures are human-owned; I did not change the JSON.
- OCCT toolkit link list in `CMakeLists.txt` (TKDESTEP/TKXSBase/TKDE/…) assumes
  the OCCT ≥ 7.8 data-exchange layout (ARCHITECTURE §4 pins ≥ 7.8; STEP moved to
  `TKDESTEP` in 7.8). Verified locally against OCCT 7.9.3; CI uses vcpkg OCCT,
  also ≥ 7.8. If a future OCCT reorg renames toolkits, update this list.
- `STEP_FIXTURE_DIR` is a compile-time absolute path baked in at configure time
  (same trade-off as `STL_FIXTURE_DIR`/`MATERIALS_JSON_PATH`): reconfigure if the
  repo is moved. A clean CI checkout always reconfigures.
- `import_step_file` requires ≥ 1 solid and ≥ 1 tessellated triangle, else throws
  `StepError`. A shell/surface-only STEP (no solid) is rejected by design — the
  voxelizer (M1.5) needs a solid.
- The quantized weld (1e-6 mm) closes these clean CAD fixtures; a STEP whose
  adjacent faces disagree on a shared edge by > 1e-6 mm would leave a boundary
  edge. Not an issue for the fixtures; revisit if a real part fails watertight.
