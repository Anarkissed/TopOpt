## What I did
Added a per-voxel passive-region design mask and a mask-aware SIMP path that
composes the existing M3.2 compliance, M3.3 filter/OC and M3.4 loop unchanged for
the unconstrained case, so every M3.1–M3.6 test is bit-identical. The mask is a
SEPARATE grid-indexed array (`DesignMask`), NOT new `VoxelTag` values, so a frozen
voxel keeps its Interior/Surface/Load/Fixture tag.

Mask type — `core/include/topopt/voxel.hpp` (extended; still Eigen-free):
- `enum class MaskValue { Active, FrozenSolid, FrozenVoid }` + `using DesignMask =
  std::vector<MaskValue>` (grid-indexed, size `voxel_count()`).
- `DesignMask make_active_mask(const VoxelGrid&)` — the all-Active base
  (implemented in the always-built `voxel/voxelize.cpp`).

Mask-aware filter — `core/include/topopt/simp.hpp` / `simp.cpp`:
- `make_density_filter(grid, radius, mask)` overload: only **Active** solid voxels
  are design/filter voxels; FrozenSolid/FrozenVoid/Empty voxels are excluded from
  every Active voxel's neighbourhood, so physical density never averages (bleeds)
  across a mask boundary. Frozen/Empty voxels get an empty CSR row (weight_sum 0).
- The existing `make_density_filter(grid, radius)` now **delegates** to the mask
  overload with `make_active_mask(grid)`. For an all-Active mask `is_active ==
  grid.solid()`, so the neighbour set/order/weights are reproduced bit-for-bit
  (verified: the MBB / gate_v2 / variants benchmarks are unchanged, plus a
  direct field-by-field equality check in the new test).

Mask-aware optimizer — `simp.hpp` / `simp.cpp`:
- `simp_optimize(grid, params, bcs, loads, options, mask)` overload. It builds an
  *effective* mask (caller mask with every Load/Fixture voxel forced to
  FrozenSolid), then:
  - **FrozenSolid**: physical density pinned to 1, design pinned to 1, excluded
    from the OC update (the internal `oc_update_masked` moves only Active voxels).
  - **FrozenVoid**: physical density pinned to 0, excluded from the design and
    from FEA stiffness. Exclusion is realised by solving on an *analysis grid* —
    a copy of `grid` with FrozenVoid voxels retagged `Empty` — so those voxels
    contribute no element and the M3.1 void-DOF gate handles any node left
    attached only to void. No FEA/assembly code changed.
  - **Active**: free design variables; the OC volume constraint targets
    `volume_fraction * (number of Active voxels)` (the Active budget), enforced on
    the filtered Active physical density.
  - The returned `design`/`physical_density` carry the pins; `volume_fraction` is
    the achieved fraction over the Active voxels. (Documented in the header:
    unlike the unconstrained overload, `physical_density` is not `filter(design)`
    at frozen voxels.)

Selection plumbing — `core/include/topopt/step.hpp` / `src/io/step.cpp`:
- `mask_step_face(grid, model, face_id, mask_value, depth_voxels, mask)` — the
  M1.6 `tag_step_face` slab selection generalized with a voxel depth. A solid
  voxel is selected iff its centre is within `(depth_voxels - 0.5)` voxel edges
  (+ epsilon) of the face's tessellation triangles (reusing the existing
  `point_tri_dist2`); `depth_voxels == 1` reproduces the M1.6 half-voxel slab. It
  writes the `DesignMask` in place and does NOT modify grid tags.

V3 group-4 retention gate turned ON — `core/tests/property/test_v3.cpp`:
- The group-4 end-to-end run now calls the mask-aware `simp_optimize` with an
  all-Active caller mask (so Load/Fixture voxels are implicitly FrozenSolid), and
  the previously reported-only Load/Fixture retention gate is now asserted
  (`gate_load_fixture_retained() && min_load_fixture_density >= 0.9`). The four
  pre-existing group-4 assertions are unchanged; the only edits are the optimize
  call, comments, and the added assertion (33 → 34 checks).

Design choices and WHY:
- **Mask type lives in `voxel.hpp`, not `simp.hpp`.** It is a per-voxel design
  classification consumed by both the optimizer (`simp/`) and the selection
  plumbing (`io/step.cpp`); voxel.hpp is the neutral header both already include,
  so no new cross-module dependency (io must not depend on the optimizer).
- **Mask behaviour is OPT-IN via an overload, not a change to the existing
  `simp_optimize`.** The M3.6 `variants` test tags Load/Fixture voxels; forcing
  implicit FrozenSolid in the existing no-mask path would have changed its
  optimized result. Keeping a separate overload leaves M3.1–M3.6 bit-identical.
- **FrozenVoid exclusion via an analysis-grid copy, not a per-voxel FEA flag.**
  "Excluded from FEA stiffness" means *no element*, not a soft rho_min element.
  Retagging FrozenVoid→Empty on a grid copy achieves exactly that through the
  existing graded `fea_solve_cg` + M3.1 void gate, with zero FEA changes.
- **Active-budget volume constraint** (`volfrac * n_active`). Makes the FrozenVoid
  test unambiguous (a forced hole + a smaller material budget both raise
  compliance) and keeps FrozenSolid "keep-in" independent of the redistributable
  budget.

### How the tests are set up and why they are trustworthy
- `passive_regions` (new, `tests/property/test_mask.cpp`, Eigen-gated): the
  24×8×8 clamped-root/tip-loaded cantilever (no voxel tags, so `effective_mask ==
  mask` — a clean comparison). (1) A FrozenVoid interior box comes out at physical
  density 0 and the constrained compliance (29.77) exceeds the unconstrained
  optimum (26.82); the constrained design is still a watertight single component
  around the void. (2) A FrozenSolid tip-top block ends at physical density 1
  exactly, and the *bite* assertion confirms the unconstrained optimum left that
  block at ≤ 0.13 (< 0.9), so the pin genuinely changed the outcome. (3) On a
  tagged cantilever with an all-Active caller mask, `check_v3` retention passes
  (min tagged density 1.000). (4) The mask-aware filter excludes a frozen voxel
  from its Active neighbours' rows while the plain filter includes it (bite), and
  the all-Active mask reproduces the plain filter field-for-field. (5) Wrong-size
  masks throw.
- `mask_face` (new, `tests/unit/test_mask_face.cpp`, OCCT-gated): drives the
  committed STEP cube fixture. `depth_voxels == 1` selects exactly the top voxel
  layer and matches the M1.6 `tag_step_face` voxel set (same count, same voxels);
  `depth_voxels == 2` selects the top two layers; the grid's tags are untouched
  (mask is a separate array); depth 0 / bad face_id / wrong-size mask throw.

## Warnings for the next run
- **The mask behaviour is only in the 6-argument `simp_optimize` overload.** The
  5-argument overload is the pre-M3.7 unconstrained optimizer and does NOT freeze
  Load/Fixture voxels — do not expect retention from it (that is why the M3.6
  `variants` test, which uses the 5-arg overload, is unaffected).
- **`result.physical_density` carries pins on the mask path**, so
  `physical_density != filter(design)` at frozen voxels and `volume_fraction` is
  the *Active* fraction (not over all solid). The unconstrained-overload
  self-consistency invariant does not hold for the mask overload by design.
- **A FrozenVoid pattern that severs the load path from the supports will throw**
  (`std::runtime_error` from the M3.1 void gate on the analysis grid) — that is
  correct (an unsupported structure has no equilibrium). Keep keep-out regions
  from fully disconnecting the loaded region from the fixture.
- **`mask_step_face` selects only solid voxels** and writes the mask in place; it
  does not clear prior selections, so a caller layering selections onto one mask
  sees them accumulate (last write wins per voxel). Start from `make_active_mask`.
- **`passive_regions` adds ~8 s** (4 × 40-iteration 24×8×8 SIMP solves +
  `check_v3`); `mask_face` is ~0.1 s. Full suite ~115 s locally, well under the
  M2.3 "< 5 min CI budget".
