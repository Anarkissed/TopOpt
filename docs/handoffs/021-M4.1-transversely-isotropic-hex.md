## What I did
Added a transversely isotropic 8-node hex element stiffness matrix alongside the
existing isotropic one, with the FDM `z_knockdown` factor weakening the
layer-normal (z) axis. Scope is the element stiffness matrix only — nothing in
the global solver, SIMP, or the isotropic path changed behaviour, and no
transverse stress recovery was added (that is M4.4's need).

Public API — `core/include/topopt/fea.hpp` (extended; still Eigen-free):
- `Hex8Stiffness hex8_stiffness_transverse(youngs_modulus, poisson,
  element_size, z_knockdown)`. Plane of isotropy = the print-layer plane (x, y);
  layer-normal axis = z. In-plane engineering constants are the full isotropic
  (E, nu) values; z is knocked down by `k = z_knockdown` in (0, 1] (the
  materials.json `z_knockdown`; resins use 1.0):
  `E_x = E_y = E`, `E_z = k·E`, `nu_xy = nu`, `G_xy = E/(2(1+nu))`,
  `G_yz = G_zx = k·G_xy`. `k = 1` collapses to `hex8_stiffness(E, nu, h)` (to
  floating-point roundoff), so isotropic mode is preserved. Same DOF order,
  Voigt convention and 2×2×2 quadrature as the isotropic element. Throws
  `std::invalid_argument` on the same material errors as `hex8_stiffness` and if
  `z_knockdown` is not in (0, 1].

Implementation — `core/src/fea/hex_element.cpp`:
- Extracted the shared Gauss integrator `integrate_hex8(const double D[6][6],
  double h)` (an anonymous-namespace helper) that computes
  `Ke = ∫ B^T D B dV` with the full 2×2×2 rule. This is a verbatim move of the
  loop that was inline in `hex8_stiffness`; both elements differ only in `D`, so
  they now share the integrator (no copy-paste of ~40 lines). `hex8_stiffness`
  is otherwise unchanged: it builds the same isotropic `D` and calls the
  integrator, so its output is bit-identical (guarded by the golden-fixture
  `hex_element` test and by V1).
- `hex8_stiffness_transverse` builds the transversely isotropic `D` and calls
  the same integrator.

Constitutive model (WHY it reduces exactly and stays positive-definite):
- Built by softening the isotropic **compliance** S (strain = S·stress) and
  inverting: the z-axial term `1/E → 1/(kE)` (so `E_z = kE`) and the two
  transverse-shear terms `1/G → 1/(kG)` (so `G_yz = G_zx = kG`); the entire
  in-plane block (x, y and their couplings, plus xy shear) is left at the
  isotropic values, so `E_x = E_y = E`, `nu_xy = nu`, `G_xy = G` are exactly
  preserved under uniaxial-stress / in-plane loading.
- The normal 3×3 compliance block factors as `(1/E)·M` with
  `M = [[1,-nu,-nu],[-nu,1,-nu],[-nu,-nu,1/k]]`. The stiffness normal block is
  written directly as the closed-form `E·M^{-1}`, whose entries algebraically
  collapse to the isotropic `D` at `k = 1` (`c(1-nu)` on the diagonal, `c·nu`
  off; `c = E/((1+nu)(1-2nu))`). `detM = (1+nu)((1-nu)/k − 2nu²) > 0` for every
  `nu ∈ (-1, 0.5)` and `k ∈ (0, 1]` (since `(1-nu)/k ≥ 1-nu > 2nu²` for
  `nu < 0.5`), so the element is SPD across the whole valid input range —
  including aggressive knockdowns like PLA's 0.55.

Design choices:
- **New sibling function, isotropic path untouched.** `hex8_stiffness` /
  `hex8_stress` keep their exact behaviour; the transverse element is opt-in.
  This is what "isotropic mode preserved; V1 must still pass" requires, and V1
  (`beam_validation`) drives only the isotropic path.
- **Shared integrator, not duplicated.** The only difference between the two
  elements is the 6×6 `D`, so the integration is factored out once. The move is
  pure (byte-identical isotropic output), verified by the golden fixture.
- **Stiffness matrix only.** M2.1's element task delivered the stiffness matrix;
  stress recovery arrived in M2.4 when the V1 gate needed it. M4.1 is the
  element; transverse stress recovery is M4.4's need (tensile stress across
  layer planes), so it is intentionally not added here.

### How the test is set up and why it is trustworthy
`hex_element` (extended, `tests/unit/test_fea.cpp`, always-built pure-C++/std
target) recovers the full 6×6 constitutive matrix `D` from an element's 24×24
`Ke` **through the public API only**: a uniform strain `eps` applied as the
nodal field `u = eps·x` is represented exactly by the trilinear element, so
`eps^T D eps = (u^T Ke u)/h³`, giving every `D_ij` by polarization
(`D_ii = q(e_i)`, `D_ij = (q(e_i+e_j) − q(e_i) − q(e_j))/2`). Then:
- (a) reconstruction sanity: the isotropic element recovers `c(1-nu)`, `c·nu`,
  `G` — validates the recovery machinery itself;
- (b) `z_knockdown = 1.0` reproduces `hex8_stiffness` entrywise (≤ 1e-9·max);
- (c) at `k = 0.55` (PLA seed): `D` is symmetric about z (`D_xx = D_yy`,
  `D_xz = D_yz`), in-plane shear `G_xy` unchanged, transverse shears
  `G_yz = G_zx = k·G`, `D_zz` strictly softer than isotropic and `D_zz < D_xx`;
  and, by inverting the recovered normal 3×3 block to the compliance, the
  engineering constants come out `E_x = E`, `E_z = k·E`, `nu_xy = nu`;
- (d) scaling law `K(E,h,k) = E·h·K(1,1,k)` (mirrors the isotropic scaling test);
- (e) argument validation: `E=0`, `nu=0.5`, `h=0`, `k=0`, `k<0`, `k>1` throw,
  and `k=1` is accepted.

## What I did NOT do
- **Did not add transverse stress recovery.** `hex8_stress` is still isotropic;
  a transversely isotropic stress path (needed for M4.4's "max tensile stress
  across layer planes, knocked down") is deferred to that task. No speculative
  API.
- **Did not wire the transverse element into `fea_solve` / `fea_solve_cg` /
  assembly / SIMP or any pipeline.** The global solvers still assemble isotropic
  `hex8_stiffness` elements. Choosing per-voxel isotropic vs transverse (and
  passing a build orientation / `z_knockdown` through the solver) is future M4
  work; M4.1 is the element only.
- **Did not read `z_knockdown` from a `Material` / materials.json.** The factor
  is a function argument; the loader (M1.2) already parses the field. Binding a
  material's `z_knockdown` into a solve belongs to the task that anisotropises
  the pipeline.
- **Did not model a build axis other than z**, and did not add a rotation of the
  layer normal. The layer-normal axis is fixed to z (grid z); orientation
  candidates that reorient the part are M4.3.
- **Did not touch `hex8_stiffness` / `hex8_stress` behaviour**, the isotropic
  `D`, the FEA assembly/solve, SIMP, marching cubes, or any M1–M3 code beyond
  extracting the shared integrator. Did not touch `/app/`, `ARCHITECTURE.md`,
  `materials.json`, CI config, any fixture, or any existing test
  assertion/tolerance. The ROADMAP change is the single M4.1 box.

## Warnings for the next run
- **The knockdown model softens `E_z` AND the transverse shears `G_yz`, `G_zx`
  by the same factor `k`; the in-plane block (`E_x`, `E_y`, `nu_xy`, `G_xy`) is
  untouched.** This is the compliance-space reading of "full in-plane
  properties, Z-direction knocked down." Note the *stiffness* entries `D_xx`,
  `D_xz` etc. do change with `k` (stiffness couples all axes) even though the
  in-plane *engineering constants* do not — verify against engineering constants
  (invert the compliance), not raw `D` entries, if you extend the model.
- **`k = 1` reduces to isotropic to floating-point roundoff, not bit-identical.**
  The transverse `D` is computed via the closed-form `E·M^{-1}`, a different
  float expression than the isotropic `c(1-nu)`/`c·nu`; the difference is
  ~1e-13 relative (the test uses a 1e-9·max tolerance). If a future test needs
  bit-identity at `k=1`, call `hex8_stiffness` directly for resins.
- **Layer normal is hard-wired to the grid z axis.** There is no orientation
  parameter yet; a part whose build direction is not grid-z is not represented
  until M4.3 introduces orientation candidates.
- **The element is SPD for the whole valid range** (`nu ∈ (-1,0.5)`,
  `k ∈ (0,1]`); the `detM > 0` proof is in the source comment. If a caller ever
  passes `k` outside (0,1] it throws rather than producing an indefinite matrix.
- **`hex_element` is unchanged in cost** (~0.04 s); the new checks are element-
  level (no solves). Full suite ~114 s locally, well under the M2.3 CI budget.
