## 2026-07-04 — M3.1 added: void-DOF safety gate before SIMP FEA

Origin: handoff 012 (M2.3 CG solver). fea_solve_cg has no singularity/null-space
guard — unlike fea_solve, which detects rigid-body modes via the SimplicialLDLT
pivot diagonal. On a consistent-but-singular system, CG converges to *a* solution
silently. SIMP (M3) will drive fea_solve_cg on density fields containing void /
near-void regions, whose unattached nodes produce zero K_ff diagonals; the Jacobi
(DiagonalPreconditioner) then divides by zero, yielding NaN or silent garbage.

Decision: added M3.1 as a hard prerequisite at the top of M3 — pin/filter zero-
diagonal free DOFs and add a null-space/zero-pivot guard to the CG path. No M3
task may call the solver before M3.1 is green. This is a ROADMAP sequencing
change only; no change to ARCHITECTURE.md §4 (solver choice unchanged) or the
Eigen-free public API. Approved by Nadim.
